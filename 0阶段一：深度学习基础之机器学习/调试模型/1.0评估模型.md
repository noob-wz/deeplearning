评估模型好坏，核心要回答三个层次的问题：

```
1. 模型在数据上表现如何？       → 指标（Metrics）
2. 模型是否真的"学到了"？       → 泛化能力（Generalization）
3. 模型在实际场景中好用吗？      → 业务/工程角度
```

### 第一层：量化指标

| 任务类型 | 常用指标 |
|---|---|
| 分类 | Accuracy, Precision, Recall, F1, AUC-ROC |
| 回归 | MSE, MAE, R² |
| 生成/NLP | BLEU, ROUGE, Perplexity |
| 检测 | mAP |


### 第二层：泛化能力
- 训练集 vs 验证集 vs 测试集的表现差距
- 是否**过拟合**（训练好但测试差）或**欠拟合**（都很差）
- 交叉验证（Cross-validation）:观察模型在**训练集 (Training set)** 和 **验证集 (Validation set)** 上的 Loss 曲线。
   * 如果训练集误差极低，但验证集误差很高，说明发生了**过拟合 (Overfitting)**（模型在死记硬背）。
   * 一个好模型应该在未见过的数据（测试集/现实数据）上表现稳定。


### 第三层：实际考量
- 推理速度、模型大小与资源消耗
    - 模型做出一次预测需要多长时间？（例如自动驾驶中的目标检测模型，如果延迟超过几十毫秒，就可能导致车祸，此时速度比极限精度更重要）。
    - 模型的参数量 (Params) 是多少？
    - 运行时需要占用多少显存 (VRAM)？是否能部署到边缘设备（如手机、摄像头）上？
- 对某些错误的容忍程度（比如医疗中假阴性代价极高）


   

---

**总结：**      
评估一个模型的步骤是，首先要根据任务类型挑出**合适的量化指标**（例如：不平衡分类看 AUC 和 F1，回归看 RMSE 和 MAE），然后在训练过程中监控 **Loss 曲线**防止过拟合，最后结合**部署环境的算力限制**，在精度和速度之间找到最佳平衡点。
