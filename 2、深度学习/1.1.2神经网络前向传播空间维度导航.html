⚠️ PyTorch / TensorFlow 常用写法：Z = X @ W.T + b
这里 X 的每一行是一个样本，W.T 让「列数=神经元数」。
维度链：(batch, in) @ (in, out) + (out,) = (batch, out)
b 通过广播机制自动扩展到 batch 维度。
第四课

多层传播：维度流水线

前向传播就像工厂流水线——每一层的输出维度 = 下一层的输入维度

🏭 3层网络的维度追踪

输入 4 特征 → 隐藏层1(8) → 隐藏层2(6) → 输出层(3)

✅ 万能口诀：
a⁰ = x  (batch, 4)
z¹ = a⁰ @ W¹ᵀ + b¹  → (batch, 8)
a¹ = σ(z¹)  → (batch, 8)
z² = a¹ @ W²ᵀ + b²  → (batch, 6)
a² = σ(z²)  → (batch, 6)
z³ = a² @ W³ᵀ + b³  → (batch, 3)
ŷ = softmax(z³)  → (batch, 3)
🔑 参数统计：
W¹ 有 8×4=32 个参数 + 8 个偏置 = 40
W² 有 6×8=48 个参数 + 6 个偏置 = 54
W³ 有 3×6=18 个参数 + 3 个偏置 = 21
总参数 = 115

✅ 终极判断法：问自己这三个问题——
① 输入的两个东西「形状一样吗？」→ 一样且要标量结果 → 内积
② 我需要「每对组合」产生一个元素吗？→ 是 → 外积
③ 我要消掉某个共享维度吗？→ 是 → 矩阵乘法（本质是批量内积）
速查表

维度速查 & 常见错误

🗺️ 维度追踪口诀

① 写下每个变量的 shape
   写代码前先在旁边标注 # (batch, in_dim)
② 矩阵乘法看「内维度」
   (m, k) @ (k, n) → 两个 k 必须相等，结果是 (m, n)
③ W 的 shape = (输出, 输入) 或 (输入, 输出)
   取决于你用 W @ x 还是 x @ W
④ 偏置 b 的 shape = (输出维度,)
   永远等于「这一层的神经元数」，通过广播加到 batch 的每个样本上

🐛 Top 3 常见 Bug

Bug 1: RuntimeError: mat1 and mat2 shapes cannot be multiplied

内维度不匹配。检查：你是不是忘了转置 W？是不是 in_features 设错了？
nn.Linear(in_features, out_features) 内部存储的是 W.shape = (out, in)

Bug 2: 偏置维度不对

b 的 shape 应该是 (out_dim,) 而不是 (in_dim,)。
手动实现时常把 b 写成输入维度的大小。

Bug 3: batch 维度上做了内积

把 (batch, features) 和 (batch, features) 做了 @ 运算 → 得到 (batch, batch)。
你需要的是「每个样本独立通过同一组权重」，不是「样本之间互相算」。

📝 PyTorch 前向传播完整模板

import torch import torch.nn as nn class SimpleNet(nn.Module): def __init__(self): super().__init__() # nn.Linear 内部: W.shape = (out, in), b.shape = (out,) # 前向计算: output = input @ W.T + b self.fc1 = nn.Linear(4, 8) # W¹: (8,4), b¹: (8,) self.fc2 = nn.Linear(8, 6) # W²: (6,8), b²: (6,) self.fc3 = nn.Linear(6, 3) # W³: (3,6), b³: (3,) def forward(self, x): # x: (batch, 4) x = torch.relu(self.fc1(x)) # → (batch, 8) x = torch.relu(self.fc2(x)) # → (batch, 6) x = self.fc3(x) # → (batch, 3) 最后一层不加relu return x # 验证维度 model = SimpleNet() x = torch.randn(32, 4) # batch=32, features=4 out = model(x) print(out.shape) # torch.Size([32, 3]) ✓
