如果我们照搬普通网络的习惯，在训练外固定采样 100 个均匀网格点（比如 0.0, 0.01, ..., 0.99），神经网络会表现出一种典型的工程失效模式：它会变得极其“短视”。在数千次迭代后，网络会精准地调整权重，使得这 100 个特定坐标点上的残差为 0，但在 0.005 这个未被采样的点上，由于没有任何 Loss 去约束它，网络输出的曲线可能会产生剧烈的扭曲和震荡。

所以，**用有限的数据集去训练模型，过拟合很容易发生，而且是标准神经网络在数学上的“默认宿命”。** 既然有限数据注定导致过拟合，普通深度学习之所以能在工业界存活并大放异彩，是因为工程师们强行加入了大量**非数据层面的约束（正则化与归纳偏置）**，以及**伪造数据（数据增强）**。

我们可以把普通网络对抗过拟合的机制，像拆解齿轮一样拆成三个工程手段：

第一，**数据增强（Data Augmentation）**。既然我们只有一万张真实的猫咪图片，工程师会在训练循环内部，对图片进行随机裁剪、旋转、色彩抖动。从数学上看，这等价于在真实数据点的高维坐标周围，人为地采样生成了无数个新的“虚拟配点”。这其实就是普通网络里的“动态随机采样”。

第二，**权重惩罚（Weight Decay / L2 正则化）**。如果网络想在有限的几个离散数据点之间画出极度扭曲的过拟合曲线，它必须依靠数值极大的权重参数。我们在 Loss 函数中额外加入一项对权重大小的惩罚（如 `loss = MSE + 0.01 * sum(W^2)`）。网络为了降低整体 Loss，被迫将权重维持在较小的值，从而在数据点之间只能画出相对平滑、“僵硬”的线，这就从力学上压制了曲线的任意扭曲。

第三，**架构的归纳偏置（Inductive Bias）**。处理图像时我们不用你写的这种全连接 MLP，而是用卷积神经网络（CNN）。CNN 强制要求空间上相邻的像素共享同一个卷积核权重，这种物理结构上的锁死，极大地剥夺了网络去“死记硬背”单个像素点的自由度。

**但是，为什么这些普通网络的手段，在 PINN 中就不够用了，必须引入无限随机采样？**

因为约束的性质发生了根本改变。      
普通网络的 Loss 约束的是**函数值（Value）**：`u_pred` 必须接近 `y_true`。
PINN 的 Loss 约束的是**函数的导数（Derivative）**：`du/dx` 必须等于 `-2x`。

在数学上，一个函数在 100 个离散点上的**值**是正确的，这条曲线大概率也跑偏不到哪里去（加上正则化后）。但是，如果一个网络仅仅在 100 个离散点上的**斜率（导数）**是正确的，它完全可以构造出一条剧烈震荡的高频正弦波，恰好在这 100 个位置的切线斜率等于预期，而点与点之间的形态完全崩溃。导数约束对空间的离散化极其敏感。

> 想象一块软木板，上面画着一个坐标系。
>
> 在 **普通神经网络（值约束）** 中，你拥有 100 条真实数据。这相当于你在木板上按下了 100 个图钉。神经网络是一根有弹性的橡皮筋。优化过程就是要求**这根橡皮筋必须穿过这 100 个图钉**。因为橡皮筋有张力（对应网络中的权重衰减/正则化），它会本能地收缩绷紧。所以在两个图钉之间的空白处，橡皮筋会被拉直，它不可能自己无缘无故地绕出一个巨大的波浪。
>
> 在 **PINN（导数约束）** 中，你没有真实数据，你只有微分方程。这相当于你拔掉了所有图钉。取而代之的是，你在那 100 个位置放上了 100 根极短的“风向标”（切线斜率）。优化过程要求：橡皮筋在经过这 100 个 x 坐标点时，它的走向必须与风向标完全平行。
>
> 请注意这个物理系统的漏洞：橡皮筋不再被“钉死”在特定的高度上，而且为了在特定的瞬间迎合风向标的角度，橡皮筋完全可以在两个风向标之间的空白处，向上甩出一个巨大的抛物线，只要它在落下来经过下一个 x 坐标时，切线角度刚好对准下一个风向标即可。

幸运的是，在 PINN 中，我们不需要像普通 CV 工程师那样去“旋转裁剪”图片，或者费尽心机调优 L2 正则化参数。因为微分方程本身就是通用的真理，定义域 $[0,1]$ 内的任意一个浮点数都可以无成本地作为训练数据。**面对过拟合，PINN 采取了最暴力、最奢华的降维打击——直接提供无限的训练数据，即无限随机采样**

### 理论到代码的桥梁

**[翻译]** 理论要求我们在每一次前向传播时，都提供一批全新的坐标点，并计算网络输出对这些坐标点的偏导数。反映在代码中，我们将在 `for epoch` 循环内部调用 `torch.rand()`，并使用 PyTorch 的 `torch.autograd.grad` API 来提取偏导数残差。

**[定位]** 在下面的代码中，核心机制存在于循环内的步骤 1（采样）和步骤 3（求导）。其余部分均为标准的 PyTorch 训练脚手架。

**[预测]** 结合我们最初讨论的偏微分方程 $du/dx = -2x$。从微积分原理可知，其通解是 $u(x) = -x^2 + C$（C 为任意常数）。我们在代码中没有任何地方指定 $C$ 的值。**在执行这段代码后，如果我们在图表中绘制出最终网络输出的 $u(x)$ 曲线，并同时绘制标准抛物线 $-x^2$，这两条曲线在视觉上应该呈现什么几何关系？**

### 代码层：第一个物理约束训练循环

请创建文件 `week09/day04_first_physics_loss.py` 并运行：

```python
import torch
import torch.nn as nn
import torch.optim as optim
import matplotlib.pyplot as plt

# 1. 定义网络架构
# 这是一个标准的 MLP。我们在隐藏层使用 Tanh 激活函数，而不是 CV 中常用的 ReLU。
class MLP(nn.Module):
    def __init__(self):
        super().__init__()
        self.net = nn.Sequential(
            nn.Linear(1, 32), nn.Tanh(),
            nn.Linear(32, 32), nn.Tanh(),
            nn.Linear(32, 1)
        )
    def forward(self, x): 
        # x shape: (batch_size, 1) -> output shape: (batch_size, 1)
        return self.net(x)

torch.manual_seed(0)
model = MLP()
optimizer = optim.Adam(model.parameters(), lr=1e-3)

losses = []

print("PINN 训练：求解 du/dx = -2x，无数据，无边界条件")

# ========== 训练循环：仅有物理约束 Loss ==========
for epoch in range(5000):
    # 步骤 1：在 [0, 1] 域内动态随机采样配点。开启 requires_grad 允许梯度反传至输入。
    # x shape: (100, 1)
    x = torch.rand(100, 1, requires_grad=True)
    
    # 步骤 2：网络前向传播
    # u shape: (100, 1)
    u = model(x)
    
    # 步骤 3：计算网络输出 u 对输入坐标 x 的偏导数 (du/dx)
    # du_dx shape: (100, 1)
    du_dx = torch.autograd.grad(
        outputs=u, 
        inputs=x,
        grad_outputs=torch.ones_like(u),
        create_graph=True
    )[0]
    
    # 步骤 4：构建物理约束 Loss (残差的均方误差)
    # 物理真理是：du/dx + 2x = 0
    residual = du_dx + 2 * x
    L_PDE = (residual ** 2).mean()
    losses.append(L_PDE.item())
    
    # 步骤 5：标准的反向传播与参数更新
    optimizer.zero_grad()
    L_PDE.backward()
    optimizer.step()
    
    if epoch % 500 == 0:
        print(f"  epoch {epoch}: L_PDE = {L_PDE.item():.6e}")

# ========== 验证阶段：网络学到了什么 ==========
# 测试时不需要计算输入梯度，固定一批均匀点进行画图
x_test = torch.linspace(0, 1, 100).reshape(-1, 1)
with torch.no_grad():
    u_pred = model(x_test)

# 根据微积分推导，如果网络学到了 u(x) = -x² + C
# 那么 u(x) + x² 应该在所有 x 上都等于同一个常数 C
u_plus_x2 = u_pred + x_test ** 2

# 绘制结果
fig, axes = plt.subplots(1, 3, figsize=(15, 4))

axes[0].plot(losses)
axes[0].set_yscale('log')
axes[0].set_title('L_PDE 训练曲线')

axes[1].plot(x_test.numpy(), u_pred.numpy(), 'r-')
axes[1].set_title('u(x) 网络预测形态')

axes[2].plot(x_test.numpy(), u_plus_x2.numpy(), 'g-')
axes[2].axhline(y=u_plus_x2.mean().item(), color='k', linestyle='--', label=f'mean C = {u_plus_x2.mean().item():.4f}')
axes[2].set_title('u(x) + x² 验证 (应为水平直线)')
axes[2].legend()

plt.tight_layout()
plt.show()
```

### ⚠️ 新手容易忽略的细节

**细节1：`nn.Tanh()` 激活函数的选择**

问题：为什么这里的激活函数用 `Tanh` 而不是深度学习中最标准的 `ReLU`？
答：`ReLU(x) = max(0, x)`。在 $x>0$ 时，它的导数恒为 1；它的二阶导数恒为 0（在原点处不可导）。如果我们在 PINN 中使用 ReLU，那么网络输出 $u(x)$ 对 $x$ 的高阶导数将全部变为 0。物理偏微分方程通常包含一阶甚至二阶导数（例如热传导方程），使用 ReLU 会导致梯度图断裂或导数为 0，PDE 残差无法计算。
如果写成 `ReLU` 会发生什么：对于本例的一阶导数，网络分段线性，拟合出的抛物线会变成极其粗糙的折线，且 Loss 极难下降；如果方程包含二阶导数，训练会直接以 Loss=0 或者无法收敛而彻底失败。`Tanh` 或 `SiLU` 是平滑无限可导的，这是 PINN 的基础设施。

**细节2：`grad_outputs=torch.ones_like(u)`**

问题：在算导数时，为什么要传入一个全 1 的张量？
答：PyTorch 的 `autograd.grad` 在底层通过雅可比向量积（Jacobian-vector product）计算导数。它默认只允许对“标量”求导。但我们的 `u` 是一个形状为 `(100, 1)` 的矩阵（100 个样本并发前向传播）。为了让 PyTorch 并行计算这 100 个样本的导数，我们必须提供一个与 `u` 形状相同的权重向量（通常全设为 1），告诉引擎：“请把这 100 个点的导数按照 1:1 的权重各自算出来”。
如果省略这个参数会发生什么：代码会触发 RuntimeError: `grad can be implicitly created only for scalar outputs`。

**细节3：`create_graph=True`**

问题：算一次导数而已，为什么要强制开启 `create_graph=True`？
答：这是 PINN 中最致命的隐藏约束。我们算出的 `du_dx` 被直接放入了 `residual` 的计算中，随后我们要对整个 `L_PDE` 调用 `backward()` 去更新网络权重参数。这意味着，“计算偏导数”这个动作本身，必须作为计算图的一个节点被记录下来，PyTorch 需要能对“导数”再求导。
如果设为 False（默认值）会发生什么：`du_dx` 算完就被当成了一个孤立的常数张量，它与网络内部参数的联系被切断。随后执行 `L_PDE.backward()` 时，引擎会报错，提示没有构建包含参数的梯度计算图，网络权重根本不会更新。
