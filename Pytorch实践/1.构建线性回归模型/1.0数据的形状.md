在Python中实现 $y = wx + b$，普通的列表（List）无法胜任，因为在真实工程中，我们需要对成千上万个数据点同时进行矩阵运算，列表的计算效率极低。所以引入了张量，**支持GPU运算**和**自动求导**


#### Pytorch中的张量细节

引入张量能高效地进行矩阵计算，但是引入一个常见的问题，那就是维度对齐。因为**深度学习中的乘法一般都是@**：
$$y = x @ w + b$$
**所以需要考虑代码中数据的维度对齐**，才不会报错，从第一天写代码就要注意张量形状对齐的铁律。

下面以没有激活函数的单层、单神经元网络为例：

- **无论样本有多少个，代码中一定都要写成二维矩阵**。例如：

    ```python
    import torch
    x = torch.tensor([[1]]) # 一个样本一个特征写成二维矩阵
    ```

- **单层的神经网络，其中的特征权重写成二维矩阵，因为代表的是多少个神经元，每个神经元有多少个权重（和特征对应）。偏置写成一维向量，对应的神经元的个数**

    ```python
    import torch
    w = torch.tensor([[1.0]])
    b = torch.tensor([3.])
    ```

- **单层神经网络前向传播后预测值的维度是：[样本的数量, 神经元的数量]**

- **反向传播之后各参数的维度和原来的参数维度一致**。
