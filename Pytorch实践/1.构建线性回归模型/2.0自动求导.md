传统的机器学习进行参数更新都是进行手动求导，那么一旦网络结构发生变化，所有的求导公式都要重写。为了解决这个问题，Pytorch引入了：**自动求导机制**和**计算图**

对叶子张量进行求导一般只需要两步：
- **设置require_grad=True，明确这是计算图的起点，也称为叶子节点（叶子张量）**

    ```python
    import torch
    x = torch.tensor([[1.],[2.]])
    y = torch.tensor([2.],[5.])

    w = torch.tensor([[1.]], requires_grad=True)
    b = torch.tensor([2.])
    ```

    - 如果写成 `w = torch.tensor([[2.0]]); w = w * 1.0; w.requires_grad = True` 会发生什么：这里的乘法操作会产生一个新张量，导致最终的 w 变成计算图中的中间节点。调用 loss.backward() 后，为了节省显存，**PyTorch 会在底层自动销毁所有中间节点的梯度**。你会发现 w.grad 的值是 None。这种隐蔽的错误不会触发程序崩溃，但会导致参数永远无法更新，模型停止学习。
