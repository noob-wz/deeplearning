重点：为初学者把反向传播讲清楚，而不是只画一个会动的网络图

反向传播：从记号、前向，到梯度回传，一步一步讲明白

这版只使用 一套固定记号：采用 row-batch 约定，单个样本写成一行 x ∈ R1×3，batch 写成 X ∈ Rm×3。两层网络统一写成：Z[1]=XW[1]+b[1]，A[1]=ReLU(Z[1])，Z[2]=A[1]W[2]+b[2]，Ŷ=σ(Z[2])。

这版先修正三个容易误导初学者的点：
1. 同层神经元 要写成 z₁[1], z₂[1]，不是把 z₁ 和 z₂ 当成前后两层。
2. 线性求和 必须明确是谁和谁相乘、维度是什么。
3. 反向传播 不能一闪而过，必须按链式法则一环一环地讲。
蓝色：前向值流动
橙色：当前正在讲的局部计算
绿色：反向传播的梯度
红色：损失 / 监督信号
先把记号讲清楚。 这里最容易混淆的是：上标 [l] 表示第几层；下标 j 表示该层第几个神经元。所以 z₁[1] 和 z₂[1] 是 同一层的两个线性输出；而 z₁[2] 才是下一层的线性输出。

先统一记号

为了避免你在代码、手算、矩阵式之间来回切换脑回路，这个页面从头到尾都使用同一套写法。

单个样本（row-batch） x ∈ R^(1×3) W^[1] ∈ R^(3×2), b^[1] ∈ R^(1×2) W^[2] ∈ R^(2×1), b^[2] ∈ R^(1×1) z^[1] = xW^[1] + b^[1] → 形状 1×2 a^[1] = ReLU(z^[1]) z^[2] = a^[1]W^[2] + b^[2] → 形状 1×1 ŷ = sigmoid(z^[2])
为什么这样写更稳？
因为当 batch 大小从 1 扩到 m 时，只需要把 x 换成 X，整套式子几乎不变。
