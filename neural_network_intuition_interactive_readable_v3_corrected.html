更准确地说：所有随着样本一起流动的激活 / 线性输出张量（X、Z¹、A¹、Z²、A²）列数都是 m。参数矩阵 W¹、W² 不属于这一类。只要盯住这一点，前向和反向传播就不容易串行。

一个容易混的条件

第二站 · 铁律

三条铁律（只在本页这个约定下这样读），记住就不混乱

你以前学线性回归时，权重 θ 常被写成一个向量。到了神经网络，最核心的变化不是数学突然变了，而是把多个这样的权重向量叠成了一块矩阵。但要注意：向量朝行放还是朝列放，必须始终跟整页的矩阵约定保持一致。

梯度形状 = 参数形状
更新哪个参数，该参数的梯度就与它形状相同。向量化反向传播的自检手段。
dW¹.shape == W¹.shape ✓
和深度学习库对接时最容易混的点：
本页是 X.shape = (n_x, m)；很多库更常见的是 X_row.shape = (m, n_x)。
两者只差一个转置：X_row = X.T。 所以这里写 Z¹ = W¹ @ X + b¹，换成“样本按行”的写法，常常会变成 Z1_row = X_row @ W1.T + b1.T。 这说明很多“写法不同”其实只是约定不同，不是谁对谁错。

第四站 · 代价

代价函数 J：对 m 个样本（列）求平均

代价函数 J 是所有样本损失的平均值。这里我让输出层也用一组固定的 W²、b² 从上面的 A¹ 真正算出 A²，所以这一节和前面的隐藏层动画是连着的，不是另外换了一套数字。这里的损失函数采用的是二元交叉熵。

所有样本损失向量 L（其中每个 L(i) 是标量）
维度对应关系
变量形状含义
A²(1, m)每列 = 某样本的预测概率
Y(1, m)每列 = 某样本的真实标签
L(i)标量第 i 个样本自己的交叉熵损失
L(1, m)把全部样本损失排成一行后的损失向量
J标量np.sum(L) / m
# 二元交叉熵代价函数
J = -(1/m) * np.sum(
  Y * np.log(A2) +
  (1-Y) * np.log(1-A2)
) # → 标量
关键提醒：loss 不对应某个隐藏神经元

代价函数里的求和只对样本维度（列）进行。神经元维度（行）在输出层 W² @ A¹ 的时候已经被「压缩」成 (1, m)。所以 J 是一个标量，不是向量。

第五站 · 反向传播

梯度的形状推导：顺着维度走就不会错

反向传播最让人糊涂的地方是：「求和在哪里？」。更准确地说：参数梯度里出现的 @ Xᵀ 或 @ A¹ᵀ，本质上是在把 m 个样本对该参数的贡献加总，最后再除以 m 取平均。

梯度公式形状推导结果形状与参数形状
dZ²A² - Y在 sigmoid + BCE 组合下可化简为这一步(1, m)
dW²(1/m)*dZ²@A¹ᵀ(1,m)@(m,n₁)(1, n₁)= W² 形状 ✓
db²(1/m)*sum(dZ²,axis=1,keepdims=True)沿列(样本)求和，并保留二维列形状(1, 1)= b² 形状 ✓
dA¹W²ᵀ @ dZ²(n₁,1)@(1,m)(n₁, m)= A¹ 形状 ✓
dZ¹dA¹⊙(1-A¹²)(n₁,m)⊙(n₁,m)(n₁, m)
dW¹(1/m)*dZ¹@Xᵀ(n₁,m)@(m,n_x)(n₁, n_x)= W¹ 形状 ✓
db¹(1/m)*sum(dZ¹,axis=1,keepdims=True)沿列(样本)求和(n₁, 1)= b¹ 形状 ✓
dW¹ 形状为什么是 (n₁, n_x)？

dW¹ = (1/m) * dZ¹ @ X.T
→ dZ¹ 形状是 (n₁, m)，X.T 形状是 (m, n_x)
→ (n₁, m) @ (m, n_x) = (n₁, n_x) = W¹ 的形状 ✓

本质上：m 是被「消去的维度」，这就是「对 m 个样本的梯度求和」的矩阵语言。

为什么这里 dZ² 能直接写成 A² - Y

因为这一页采用的是 sigmoid 输出层配合 binary cross-entropy。如果你只记住结论、不记住条件，之后一换输出层或损失函数就会重新混乱。这里建议把它记成：dZ² = A² - Y 是特定组合下的化简结果。

为什么 db¹ 必须加 keepdims=True

np.sum(dZ¹, axis=1) 沿样本维度（列）求和。
db² 也同理：如果想严格保持和 b² 一样的二维形状，就应写成 keepdims=True。
如果不加 keepdims=True：结果是 (n₁,) = (3,)，是一维向量
如果加了 keepdims=True：结果是 (n₁, 1) = (3, 1)，可以正确广播

形状 (3,) 和 (3,1) 在广播时行为不同，必须保持列向量形状，否则后续计算出错。
