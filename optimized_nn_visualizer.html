统一约定
本组件统一采用 batch 按行 的写法：Z = XW + b。因此：X ∈ ℝm×n_in，W ∈ ℝn_in×n_out，b ∈ ℝ1×n_out。
这样做的好处是：单样本、批量、前向传播、反向传播、矩阵化可以全部使用同一套方向感，不会在 Wx+b 和 XW+b 之间来回切换。

步骤 0 / 4

一个神经元，本质上就是“加权求和 + 激活”

把输入、权重、偏置、求和节点、激活节点拆开看，就会发现“神经元”并不是一个黑盒圆点，而是一段很短但结构完整的计算过程。

当前重点
避免混淆
线性部分输出的是 z，激活后输出的是 a。这两个量不要混在一起。前者决定“原始打分”，后者决定“送往下一层的非线性表示”。
点击播放，观察输入如何一路向右流到输出和损失。

前向传播：按层计算值

这里展示的是同一套 row-batch 约定下的两层网络：先得到隐藏层线性输出 Z₁ = XW₁ + b₁，再经过激活得到 A₁，最后得到输出 Ŷ 与损失。

当前重点
先分清：线性输出 Z 与激活输出 A 是两回事。
Z₁ = XW₁ + b₁ A₁ = ReLU(Z₁) Z₂ = A₁W₂ + b₂ Ŷ = sigmoid(Z₂) L = loss(Ŷ, Y)
避免混淆
这一步只负责 算值，不负责更新参数。参数更新永远发生在反向传播之后。
点击播放，观察梯度如何从损失反向回流。

反向传播：按链式法则回传梯度

从损失开始，先拿到输出层误差信号，再把梯度传回隐藏层。你看到的不是“一个神秘算法”，而是在同一张图上把方向反过来。

当前重点
先理解梯度方向：它永远从损失回流到更早的参数。
δ₂ = ∂L / ∂Z₂ ∂L/∂W₂ = A₁ᵀδ₂ δ₁ = (δ₂W₂ᵀ) ⊙ ReLU'(Z₁) ∂L/∂W₁ = Xᵀδ₁ W ← W - α · ∂L/∂W
避免混淆
梯度 不是参数更新。梯度只告诉你“往哪个方向调整更可能降低损失”，真正更新参数的是优化器那一步。

矩阵化：把“很多个神经元 × 很多条样本”合成一次乘法

矩阵化不是换一种写法而已，它是把整层所有神经元、整个 batch 所有样本的线性运算，压缩成一次高效的矩阵乘法，方便并行执行。

当前重点
只记一句：内维匹配，外维保留。因此 X(m×n_in) · W(n_in×n_out) = Z(m×n_out)。
单样本（按行写）: z = xW + b, x ∈ ℝ¹ˣⁿⁱⁿ, W ∈ ℝⁿⁱⁿˣⁿᵒᵘᵗ 批量: Z = XW + b, X ∈ ℝᵐˣⁿⁱⁿ, W ∈ ℝⁿⁱⁿˣⁿᵒᵘᵗ, Z ∈ ℝᵐˣⁿᵒᵘᵗ 反向: ∂L/∂W = Xᵀδ, ∂L/∂X = δWᵀ, ∂L/∂b = sum(δ, axis=0)
避免混淆
不要在同一页同时混用 Wx+b 和 XW+b。只要你固定一种方向，所有维度推断都会变得稳定。
