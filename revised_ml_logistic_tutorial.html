第一章

线性回归：先理解“连续数值拟合”这件事

线性回归的原始任务，是用一条直线去拟合连续数值。模型本体非常简单：

ŷ = wx + b
// ŷ：预测的连续值，w：斜率，b：截距

训练的真正问题不是“画条线”，而是：什么叫画得好？ 一旦定义了损失函数，训练就变成了调 w 和 b，让损失下降。

🎯 核心顿悟：在线性回归里，真正直接进入梯度的是 (ŷ−y)。再乘上 x 之后，才变成某个样本对参数 w 的实际推动量。也就是说：对输出的误差 和 样本所在的位置，两者共同决定更新。
先把链式法则记牢：
对单个样本，线性回归里对权重的梯度是 ∂L/∂w = (∂L/∂ŷ) · (∂ŷ/∂w)。
若用平方误差，∂L/∂ŷ = 2(ŷ−y)；而 ∂ŷ/∂w = x。
所以最终真正作用到参数上的，是 2(ŷ−y)x，而不是只有“几何距离”这一个量。
第二章

为什么线性回归不适合做分类？

这里要特别小心：“线性回归不适合分类”不是只因为异常值。它至少有三层原因，而且三层原因不应该被一张图混着解释。

拆开看：左边说明“输出越界”，右边说明“异常值会拖偏阈值”
0.00
🎯 读图方式：左半图不是在说异常值，而是在说明 ŷ=wx+b 本体就是无界实数，所以它会自然跑出概率区间 [0,1]；右半图才是在说明：如果你硬把线性回归用于分类，再额外规定 ŷ=0.5 当阈值，那么远处极端点会强烈拖偏这条“人为阈值”。
线性回归不适合分类，原因分三层：
1. 输出层面：ŷ=wx+b 是任意实数，可能小于 0 或大于 1，不能天然解释成概率。
2. 优化/鲁棒性层面：平方误差会放大远点，极少数异常样本可能主导更新。
3. 建模目标层面：分类真正在乎的是“给真实类别多大概率”，而不是把 0/1 当普通连续数值去逼近。
设计者接下来的问题：
有没有一个函数，能把任意实数先变成 (0,1) 内的概率；并且还能给出一个天然阈值，让“分类边界”不是后加规则，而是模型结构的自然结果？——这就是 Sigmoid 的出场理由。
第三章

从特征 x 到线性得分 z，再到概率 p 的完整因果链

这一章专门修复最容易混的地方：sigmoid 图上的 z=0，和特征空间里的 wx+b=0，并不是两条互相竞争的解释，而是同一件事在不同空间中的投影。

// 第一步：先做线性组合
z = wx + b

// 第二步：再把 z 送入 sigmoid 得到概率
p = σ(z) = 1 / (1 + e−z)

// 第三步：若用 p=0.5 作为分类阈值
p = 0.5 ⇔ z = 0 ⇔ wx + b = 0

🎯 核心顿悟：真正先发生的，是在特征空间里把 x 映射成 z；sigmoid 只是把这个 z 翻译成概率。决策边界不是“从 sigmoid 曲线里长出来”的，而是条件 z=0 投回特征空间后得到的 wx+b=0。
必须严格区分两个函数：
· 若横轴是 z，你画的是 p = σ(z)。这条 sigmoid 曲线本身不会因为 w、b 改变。
· 若横轴是 x，你画的是 p(x) = σ(wx+b)。这时 w、b 会改变整条曲线的形状与位置。

所以“改变 w 会让 sigmoid 更陡”这句话，只有在你说的是 p(x)=σ(wx+b) 的 x-空间图像时才成立；如果横轴是 z，那条 sigmoid 根本没变。
改变 w 和 b 分别在做什么？
· 改变 w：先改变 x 映射到 z 的缩放和方向；在 x-空间里表现为 p(x)=σ(wx+b) 更陡、更缓，甚至左右翻转。
· 改变 b：改变整体的 z 偏移；在 x-空间里表现为整条概率曲线左右平移。
· 决策边界始终由 wx+b=0 决定；一维时就是一个点 x*=−b/w，画在数轴上是竖线，不是 sigmoid 图里随意挑出来的点。
第四章

逻辑回归：把几块零件拼成一个完整模型

到了这里，逻辑回归已经不该再被理解成“线性回归外面套了个 sigmoid”。更准确地说，它是在做这样一件事：先用线性组合得到证据强度 z，再用 sigmoid 把它解释成概率，然后用交叉熵去衡量这个概率与真实标签的匹配程度。

交叉熵损失：惩罚的是“错误概率态度”

L = −[y·log(p) + (1−y)·log(1−p)]
// y=1 时：L = −log(p)
// y=0 时：L = −log(1−p)
这里一定要说完整：
“−log(p)” 只在 y=1 时成立；“−log(1−p)” 只在 y=0 时成立。
完整写法必须是上面的统一形式，因为逻辑回归要同时处理正类和负类。

链式法则：概率误差和参数梯度不是一回事

\[ \frac{\partial L}{\partial w} = \frac{\partial L}{\partial p} \cdot \frac{\partial p}{\partial z} \cdot \frac{\partial z}{\partial w} \]
在逻辑回归里，这三层最后会化简成：
∂L/∂w = (p − y) · x
术语要分开：
· 概率误差：p−y —— 表示模型给类别1的概率，和真实标签之间差了多少。
· 样本对参数 w 的梯度贡献：(p−y)x —— 这才是某个样本最终把权重往哪个方向推、推多大的量。

所以，“影响大小”绝不能只说成由代价函数对预测值的导数决定；它还必须继续乘上 预测值对参数的导数，也就是链式法则的后半段。

🎯 核心顿悟：这里不能说“力度只取决于 p−y”。更准确地说：p−y 决定概率层面的错得多不多，再乘上 x，才变成对参数 w 的真实贡献。也就是说，概率误差 和 样本所处位置，两者共同决定更新。
和线性回归“按距离拉”的根本区别：
· 在线性回归里，若用平方误差，∂L/∂ŷ 直接随着数值误差放大。
· 在逻辑回归里，真正作用到参数上的量是 (p−y)x。
· 因此它不是由“几何距离”直接线性决定的，而是由“概率错多少 × 这个样本所在的位置”共同决定。
· 几何距离可能间接影响 z 和 p，但它不再像线性回归里那样，直接就是惩罚本体。

最后用一句话收束：
线性回归关心的是“这个数值差了多少”；逻辑回归关心的是“我给真实类别分配了多大概率”。
两者表面上都写成 wx+b，但因为后面的激活与损失不同，整个模型的意义、梯度和学习行为都变了。
