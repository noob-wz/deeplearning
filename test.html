第一章

线性回归：用一根木棍拟合散点

想象你收集了一批数据：房子的面积和它的价格。把这些点画在坐标系里，你会发现它们大致沿着一条斜线分布。线性回归的任务，就是找到那根"最合适"的线。

设计者的第一个问题：什么叫"最合适"？我们需要先定义一个衡量标准，才能谈"最优"。这个标准就是损失函数。

模型长什么样

线性回归的预测非常直白，就是一条直线方程：

ŷ = w · x + b
// ŷ = 预测值，x = 输入特征，w = 权重（斜率），b = 偏置（截距）

这里只有两个参数需要学习：w（斜率，决定直线倾斜角度） 和 b（截距，决定直线上下位置）。整个训练过程就是找到最好的 w 和 b。

比喻解释：把每个样本点当作钉在木板上的图钉，直线是一根木棍，每个图钉和木棍之间用弹簧相连。弹簧越长（误差越大），拉力越大。木棍会在所有弹簧的合力下自动旋转、平移，直到"所有弹力之和最小"——这就是模型训练的本质。

MSE vs MAE 的区别：

· MSE（均方误差）下：弹力的大小（模型对单个样本的敏感度） = Loss对预测值的导数 = 2(预测值-真实值) ，离得远的点惩罚急剧增大，对异常值非常敏感。

· MAE（平均绝对误差）下：弹力的大小（模型对单个样本的敏感度） = Loss对预测值的导数 = 正负1，所有样本点对模型的拉力一致，对异常点的抗干扰能力强

工程上默认用 MSE，因为它处处可导，梯度下降更顺畅。
// MSE 损失函数
L = (1/n) · Σ (ŷᵢ - yᵢ)²

// 展开：预测值 ŷ = wx + b
L = (1/n) · Σ (wxᵢ+b - yᵢ)²

🎯 梯度下降：怎么调整木棍

有了损失函数，下一个问题是：如何系统地调整 w 和 b 让损失变小？

答案是对损失函数求导。导数告诉我们：当前 w 和 b 在"损失地形"的哪个方向上是下坡方向。沿着下坡方向走，损失就会减小——这就是梯度下降。

// 对 w 求偏导（"弹力 × 特征 x" 的直觉）
∂L/∂w = (2/n) · Σ (ŷᵢ - yᵢ) · xᵢ

// 对 b 求偏导（只是弹力之和）
∂L/∂b = (2/n) · Σ (ŷᵢ - yᵢ)

// 更新规则（α = 学习率）
w ← w - α · ∂L/∂w
b ← b - α · ∂L/∂b
根据链式法则（针对所有模型）

关于 w 的导数 = "L 关于 ŷ 的导数" 乘以 "ŷ 关于 w 的导数"。所以在线性回归模型下：
  • L 关于 ŷ 的导数 → 就是弹力的大小（模型对单个样本点的敏感度）
  • ŷ 关于 w 的导数 → 就是特征 x（力矩）

所以，单个样本对模型最终的影响（真实破坏力） = 对权重的导数 = 弹簧弹力大小(L 关于 ŷ 的导数) * 力矩(x)。

总结就是：一个样本点对模型的最终影响，不仅取决于它错得有多离谱（弹簧拉力），还取决于它站在什么位置（杠杆有多长）！>

第二章

为什么线性回归不能做分类？

假设我们要判断一封邮件是否是垃圾邮件（是=1，否=0）。如果直接用线性回归预测 y，会发生什么？

👆 把滑块拉到右边，加入一个极端点——观察原本不错的分类线如何被破坏

线性回归用于分类的三个致命问题：
1. 预测值超出 [0,1]：线性回归可以输出 -5 或 3.7，而概率必须在 0 到 1 之间
2. 对异常值极度敏感：一个极端点会把整条线拉偏，破坏所有已经分好的类
3. 损失函数不匹配：分类问题是"对了就是对了，错了就是错了"，不适合用数值大小来惩罚
设计者的思路：我们需要一个函数，能把任意实数 z 压缩到 (0,1) 范围内，而且在 z=0 处有一个自然的分界点。有没有这样的函数？——有，它叫 Sigmoid。
第三章

Sigmoid：把任意数值压缩成概率

σ(z) = 1 / (1 + e-z)

// z 可以是任意实数：-∞ 到 +∞
// σ(z) 的输出：永远在 (0, 1) 之间
// σ(0) = 0.5 ← 这是关键：z=0 是自然分界点

拖动滑块感受：z 越大 → 概率越接近 1；z 越小 → 概率越接近 0；z=0 → 概率恰好 0.5

为什么 z=0 是决策边界？

在逻辑回归里，我们让 z = wx + b，然后送入 sigmoid：p = σ(wx+b)。

我们规定：当 p ≥ 0.5 时，预测为类别 1；当 p < 0.5 时，预测为类别 0。

那么分界点在哪？p = 0.5 ⟺ σ(z) = 0.5 ⟺ z = 0 ⟺ wx + b = 0

关键顺序（你说的"果而非因"✓）：
先有 sigmoid 函数 → 用 p=0.5 作阈值 → 对应 z=0 → 投影回特征空间得到 wx+b=0 这条线。
wx+b=0 是决策边界，它是 sigmoid 分类规则的自然结果，不是我们强行规定的。
第四章

逻辑回归：完整拼图

现在我们把所有零件组装起来：

逻辑回归完整流程图
// 步骤 1：把所有特征线性组合成一个值 z
z = w₁x₁ + w₂x₂ + ... + wₙxₙ + b

// 简写为向量形式
z = wᵀx + b

z 可以是任何实数。它综合了所有特征的"证据"：z 越大，说明特征越支持"是类别1"；z 越小，说明越支持"是类别0"。

多个特征时，w 是一个向量，每个 wᵢ 代表对应特征 xᵢ 在判断中的"重要程度"和"方向"。

// 步骤 2：把 z 转换为概率
p = σ(z) = 1 / (1 + e⁻ᶻ)

// p 代表"样本属于类别1的概率"
// 决策：p ≥ 0.5 → 类别1；p < 0.5 → 类别0

sigmoid 不只是"压缩"了数值，它有更深的含义：在统计上，sigmoid 正是二元分类问题在指数族分布假设下的自然输出函数（这是为什么它"刚好"好用的深层原因）。

// 步骤 3：损失函数——交叉熵
// 单个样本损失：
L = -[y·log(p) + (1-y)·log(1-p)]

// 当 y=1（真实是正例）：L = -log(p)
// 当 y=0（真实是负例）：L = -log(1-p)

// 整个数据集的平均损失：
J = (1/n) · Σ Lᵢ
交叉熵损失的行为
⚠️ 你之前只写了 -log(p)，这只对 y=1 时成立！
完整形式必须考虑两种情况：y=1 用 -log(p)；y=0 用 -log(1-p)。
合并写法 -[y·log(p) + (1-y)·log(1-p)] 通过 y 的取值自动"切换"——这是一个优雅的数学技巧。
为什么不用 MSE？
逻辑回归如果用 MSE，损失函数关于 w 的图像是非凸的（有很多局部最小值），梯度下降很容易卡住。交叉熵保证了损失曲面是凸的（有唯一全局最小值）。这是选择交叉熵的数学原因。
// 步骤 4：梯度下降更新（形式与线性回归惊人地相似！）
∂J/∂w = (1/n) · Σ (pᵢ - yᵢ) · xᵢ

∂J/∂b = (1/n) · Σ (pᵢ - yᵢ)

w ← w - α · ∂J/∂w
b ← b - α · ∂J/∂b
漂亮的对称性：逻辑回归的梯度公式和线性回归几乎一样，都是"误差 × 特征"。唯一区别：线性回归的"误差"是 (ŷ-y)，逻辑回归的"误差"是 (p-y)。这不是巧合，而是因为它们都是广义线性模型 (GLM) 的特例——深层数学结构是一致的。

第五章

对比总结：两者的本质区别

线性回归

项目内容
任务预测连续数值
输出任意实数 ŷ
模型ŷ = wx + b
损失MSE: (ŷ-y)²
激活无（恒等函数）
比喻弹簧拉木棍找平衡

逻辑回归

项目内容
任务预测类别（分类）
输出概率 p ∈ (0,1)
模型p = σ(wx + b)
损失交叉熵: -[y·log(p)+...]
激活Sigmoid
比喻错误分类时弹力无穷大

关键洞见：它们的共同本质

两者都是"广义线性模型 (GLM)"
核心都是 z = wx + b，区别只是最后套了什么"激活函数"：
· 线性回归：套了"恒等函数"（不做变换）
· 逻辑回归：套了"sigmoid 函数"
· 神经网络：套了各种激活函数（ReLU、tanh 等）……是的，神经网络就是这个思路的扩展。

梯度公式的对称美

// 线性回归
∂L/∂w = (1/n) · Σ (ŷᵢ - yᵢ) · xᵢ

// 逻辑回归（形式完全一样！只是误差定义不同）
∂J/∂w = (1/n) · Σ (pᵢ - yᵢ) · xᵢ

// 统一形式：梯度 = 误差 × 特征，这是最重要的直觉
🎯 最后记住这一句话：
训练神经网络 = 找一组参数 w 和 b，让损失函数最小。不管模型多复杂，都是这一件事。 梯度下降就是算"向哪个方向调整参数能让损失最快降低"，然后迈一小步。重复这个过程，直到收敛。

Python 代码对照（验证你的理解）

import numpy as np

# ── 线性回归 ──
def linear_regression_step(X, y, w, b, lr):
  y_hat = X @ w + b            # 前向传播
  error = y_hat - y            # 误差（弹力）
  dw = (error @ X) / len(y)   # 梯度 = 误差 × x
  db = error.mean()
  return w - lr*dw, b - lr*db

# ── 逻辑回归 ──
def sigmoid(z): return 1 / (1 + np.exp(-z))

def logistic_regression_step(X, y, w, b, lr):
  z = X @ w + b               # 线性组合
  p = sigmoid(z)              # 转概率
  error = p - y               # 误差（注意：p 不是 z）
  dw = (error @ X) / len(y)   # 梯度形式完全一样！
  db = error.mean()
  return w - lr*dw, b - lr*db
