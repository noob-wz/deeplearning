# 12 个月执行地图

我按你之前的节奏来设计：
**每天 1.5–2 小时，周末稍微多一点。**

---

## 第 0 月（可选，2 周补丁）

**适用人群：Python / NumPy / Git 还不稳**

### 目标

把“能看课”变成“能自己动手”。

### 你要做到

* 能自己写 Python 脚本
* 能读写 NumPy 数组
* 会用 matplotlib 画 loss 曲线
* 会用 Git 提交代码
* 会在 Colab / 本地跑 PyTorch

### 输出

* 1 个 `numpy_basics.ipynb`
* 1 个 `pytorch_basics.ipynb`
* 1 个 GitHub 仓库

### 为什么必须有这一步

fast.ai 官方本身就写明它面向的是“**有一些 coding experience**”的人；所以如果这块虚，你后面会一直“能跟着跑、但不会独立做”。

---

## 第 1–2 ([Practical Deep Learning for Coders][1])课：全部

* 李宏毅 Lecture 1
* 李宏毅 PyTorch 相关补充

### 目标

你要真正吃透：

* 损失函数
* 梯度下降
* 过拟合 / 欠拟合
* 正则化
* 前向传播 / 反向传播直觉
* 用 PyTorch 写最小训练循环

### 项目

做一个最小图像分类项目：

* 数据：MNIST 或 CIFAR-10
* 模型：MLP + 简单 CNN
* 你自己画：

  * train loss
  * val loss
  * accuracy 曲线

### 验收

你要能不看资料回答：

* 为什么会过拟合？
* 为什么验证集比训练集差？
* Adam 比普通梯度下降好在哪？

### 备注

这一段继续沿用你原来的安排，非常好，不需要大改。

## 第 3 月：优化、调参与误差分析

### 主线

* 李宏毅 Lecture 2
* 吴恩达第二门课里你原来保留的重点内容
* d2l 对应章节查缺补漏

### 目标

你要建立这几个能力：

* 看 loss 曲线判断问题
* 理解 Adam / BN / 正则化的作用
* 学会最基础的误差分析
* 学会做一个规范实验

### 项目

升级前面的 CIFAR-10 项目：

* 加数据增强
* 改优化器
* 改学习率
* 做一个简短实验报告

### 输出

GitHub 上新增一个仓库或子目录：

* `baseline`
* `adam_vs_sgd`
* `augmentation_ablation`
* `error_analysis.md`

### 这一阶段后你应该具备的状态

不是“我听过 CNN”，而是：

> 我已经会训练一个小模型，并且知道训练坏了以后怎么诊断。

DLS 官方到现在仍把模型评估、误差减少、CNN 基础作为核心结果；d2l 也仍是交互式代码+数学结合的参考，非常适合这个阶段随查随用。([Coursera][2])ture 3（CNN）

* d2l 的 CNN 相关章节
* Fast.ai Part 1 中与你图像任务最相关的部分

### 目标

你要真正掌握：

* 卷积、padding、stride
* 特征图尺寸变化
* 为什么 CNN 比全连接更适合图像
* 迁移学习基本用法

### 项目

做一个“像样一点”的图像分类项目：

* 自选小数据集
* 用预训练模型 fine-tune
* 输出 confusion matrix
* 做错误样本可视化

### 输出

这是你第一个能放简历的项目雏形。

### 为什么这里引入 Fast.ai

fast.ai 的优势依然是**快速做出可运行结果**，而且官方仍提供课程和配套书；但它更适合作为提速器，不适合作为你唯一的理论主线。([Practical Deep Learning for Coders][1])会强制加进来。

### 主线

* OpenCV 基础
* 相机模型
* 相机标定
* 投影关系
* 基础位姿估计直觉

### 目标

你要能理解：

* 内参是什么
* 外参是什么
* 为什么会有畸变
* 标定在车载视觉里为什么是基础设施

### 项目

做 1 个传统视觉项目：

* **车道线检测**

  * 透视变换
  * 边缘检测
  * 曲线拟合
* 或 **相机标定实验**

  * 棋盘格拍摄
  * 求内参
  * 畸变矫正

### 输出

* `camera_calibration.ipynb`
* `lane_detection_demo.ipynb`
* 1 篇博客：
  《为什么汽车视觉不只是 YOLO》

### 为什么这一块必须补

OpenCV 官方把 `calib3d` 单独作为核心模块，直接覆盖 camera calibration、pose estimation、3D reconstruction；Autoware 官方教程里也把 intrinsic camera calibration、lidar-camera calibration、lidar-IMU calibration 明确列为教程主项。([OpenCV 文档][3])测基础：IoU、NMS、mAP

* 语义分割基础：mask、pixel accuracy、mIoU
* 继续强化 PyTorch 工程能力

### 项目

你需要做两个项目中的至少两个：

#### 项目 A：交通目标检测

* 数据可先用通用数据集
* 后续迁移到自动驾驶数据集
* 模型不求最先进，但要完整
* 重点是：

  * 数据处理
  * 训练
  * 评估
  * 误检分析

#### 项目 B：道路/车道/可行驶区域分割

* 可以做二分类或多分类
* 强调可视化和失败案例分析

#### 项目 C：交通灯检测或分类

* 规模可以小
* 但很有“汽车味”

### 输出

你要开始形成“作品集三件套”的前两件：

* 检测项目
* 分割项目

### 这一步的定位

从这里开始，你简历上的项目就不该再是“手写 MNIST”那种学生作业感了，而要开始像行业项目。

## 第 8 月：进入自动驾驶数据集

### 主线

* Waymo Open Dataset 或 nuScenes
* 学会看自动驾驶数据标注格式
* 学会读取多传感器数据结构

### 目标

你要从“普通 CV 项目”过渡到“汽车感知项目”。

### 项目

从下面选 1 个：

* 自动驾驶场景 2D 检测
* 多摄像头感知可视化
* 基于标注的时序目标跟踪入门
* 用一个小子集做数据分析 + baseline

### 为什么这一步很重要

Waymo Open Dataset 官方仍公开提供自动驾驶感知数据，包含多路相机和激光雷达等多传感器内容；官方介绍写到感知数据集中有 **5 路相机** 和顶置激光雷达相关对象资产。Autoware 也持续提供用于测试和开发的数据入口。([Waymo][4])基本概念

* node / topic / service
* package 创建
* Python 节点
* 能读懂基础 C++ 节点

### 目标

你要能把“模型”开始放进“系统”。

### 项目

做 1 个 ROS 2 小项目：

* 订阅图像 topic
* 做简单图像处理
* 发布处理后的结果
* 或做一个最简单的检测结果发布节点

### 为什么这一步一定要有

ROS 2 官方教程明确就是按顺序逐步建立技能；Autoware 的运行时本身也建立在 ROS 之上。对汽车/机器人视觉岗位来说，你不能只会 notebook。([ROS 文档][5])

* 推理测速
* TensorRT 基础概念
* 模型压缩/速度-精度权衡

### 项目

把你前面的一个检测或分割模型做一次轻量部署：

* 导出 ONNX
* 比较 PyTorch vs ONNX 推理速度
* 能的话进一步接触 TensorRT

### 输出

* `deployment_report.md`
* FPS、latency、精度对比图

### 为什么这一步很值钱

Autoware 现有官方文档里既有基于 YOLOX 的图像检测包，也有把 ONNX 交给 `trtexec` 构建 TensorRT engine 的相关说明；这说明“训练完模型”不是终点，车端落地默认要考虑推理链路。([Autoware Foundation][6])选一个方向深入一点：

* 多目标跟踪
* 简单传感器融合
* 图像 + 点云投影关系
* 时序稳定性处理

### 项目

推荐做其中一个：

#### 方向 1：Tracking

* 检测结果 + 简单跟踪
* 输出 ID 稳定展示

#### 方向 2：Image-LiDAR 对齐可视化

* 把点云投到图像上
* 讲清楚外参与投影关系

#### 方向 3：2D 检测 + 3D/雷达信息简单关联

* 哪怕只做可视化和最小 demo 也很好

### 为什么这一层是“汽车味”的关键

Autoware 官方能力范围里明确覆盖 perception、multi-sensor fusion、tracking，也有 image projection based fusion、detection by tracker、radar 关联等模块。你不一定要做很深，但一定要沾到。([Autoware Foundation][7])事：

### 1. 整理 3 个代表项目

我建议最终作品集长这样：

#### 项目 1：交通目标检测

* 有训练、评估、误差分析
* 有可视化
* 有失败案例

#### 项目 2：道路/车道/可行驶区域分割

* 强调场景理解
* 强调业务意义

#### 项目 3：几何/ROS/部署相关项目

三选一：

* 相机标定 + 投影
* ROS 2 图像节点 + 推理
* ONNX/TensorRT 部署实验

### 2. 补简历叙事

每个项目都能回答这 6 个问题：

* 任务是什么
* 数据是什么
* 模型是什么
* 指标是什么
* 最难的问题是什么
* 你是怎么改进的

### 3. 面试准备

你至少要能清楚讲这几类题：

* BN / Adam / 正则化
* 过拟合 / 欠拟合 / 学习曲线
* IoU / NMS / mAP
* 语义分割指标
* 相机内参/外参
* 为什么要标定
* ROS 2 基本概念
* 模型部署为什么要考虑延迟

# 你每个月的时间分配

我建议固定成这个比例：

* **30% 做项目**
* **20% 写总结 / 复盘 / GitHub 整理**

# 最该避免的 3 个问题

## 1. 不要把 d2l 当成从头到尾必须线性读完的大书

它更适合当参考书。官方现在仍把它定位成交互式深度学习书，适合边做边查。

## 2. 不要把 Fast([深度学习入门][8])位仍是给已有一些编码经验的人做 practical deep learning。

## 3. 不要等“学完整套([Practical Deep Learning for Coders][1])有没有听过 ResNet”，而是：

* 标定
* 数据格式
* 系统接入
* 实时性
* 失败案例分析
