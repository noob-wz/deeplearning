## 关键代码行逐一讲解

---

### 代码行 1：`x` 和 `w` 从标量变成向量

#### ① 理论问题

一个样本有多个特征，每个特征需要有自己的权重。
如何用一个统一的数学结构表示“多个特征”和“多个权重”？

#### ② 理论困境

标量 `w` 乘以标量 `x`，只能处理一个特征。
处理多个特征要写成多行：

```text
pred = w1*x1 + w2*x2 + w3*x3
```

如果有 1000 个特征，就无法扩展。

#### ③ 理论解法

把多个特征打包成向量：

```text
x = [x1, x2, x3]
w = [w1, w2, w3]
```

用点积代替逐项相乘再求和：

```text
w · x = w1×x1 + w2×x2 + w3×x3
```

#### ④ 代码实现

```python
import torch

# 一个学生的三个特征
x = torch.tensor([8.0, 7.0, 0.9])

# 三个权重，每个特征一个
w = torch.tensor([0.6, 0.3, 0.1], requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)

print(x.shape)   # torch.Size([3])
print(w.shape)   # torch.Size([3])
```

#### ⑤ 手动验证

手算点积：

```text
w · x = 0.6×8.0 + 0.3×7.0 + 0.1×0.9
      = 4.8 + 2.1 + 0.09
      = 6.99
```

代码验证：

```python
manual = 0.6 * 8.0 + 0.3 * 7.0 + 0.1 * 0.9
print(f"手算: {manual:.2f}")

pred = torch.dot(w, x) + b
print(f"代码: {pred.item():.2f}")   # 应该一致 ✓
```

#### ⑥ 破坏实验

```python
# x 有 3 个元素，w 有 4 个元素，会怎样？
w_bad = torch.tensor([0.6, 0.3, 0.1, 0.5], requires_grad=True)

try:
    torch.dot(w_bad, x)
except RuntimeError as e:
    print(e)   # 读报错，解释为什么
```

### 代码行 2：`torch.dot(w, x)`（点积）

#### ① 理论问题

多个特征各有自己的重要程度，
需要一个运算把“每个特征 × 对应权重”全部加起来，得到一个预测值。

#### ② 理论困境

逐项手写：

```text
w1*x1 + w2*x2 + …
```

在代码上不可扩展，在数学上也不简洁。

#### ③ 理论解法

向量点积：

```text
w · x = Σ(wᵢ × xᵢ)
```

结果是一个**标量**，正好是我们想要的预测值。

#### ④ 代码实现

```python
y_pred = torch.dot(w, x) + b
print(y_pred)
# 一个标量，shape 是 torch.Size([])
```

#### ⑤ 手动验证

展开验证：点积等于逐项相乘再求和。

```python
manual_dot = (w * x).sum()
torch_dot = torch.dot(w, x)

print(f"手动展开: {manual_dot.item():.4f}")
print(f"torch.dot: {torch_dot.item():.4f}")   # 相同 ✓
```

点积有交换律吗？

```python
print(torch.dot(w, x).item())
print(torch.dot(x, w).item())   # 相同，点积可交换
```

#### ⑥ 破坏实验

```python
# 用矩阵乘法代替点积
# w @ x 和 torch.dot(w, x) 一样吗？
print(w @ x)           # 可以，shape []
print(torch.dot(w, x)) # 一样

# x @ w 呢？
print(x @ w)           # 也可以，因为都是 1D

# 如果 w 是 [1, 3] 而不是 [3]，@ 的行为怎么变？
w2d = w.unsqueeze(0)   # shape [1, 3]
print(w2d @ x)         # shape [1]，不再是标量
```

让学生解释 `shape` 的变化。

### 代码行 3：`w.grad` 的 shape 和 `w` 一样

#### ① 理论问题

`w` 是一个向量 `[w1, w2, w3]`，
每个 `wᵢ` 都影响 `loss`，
所以每个 `wᵢ` 都有自己的梯度 `∂loss/∂wᵢ`。

#### ② 理论困境

梯度不再是一个数，而是多个数——
一个数对应一个权重。
这些梯度怎么组织？

#### ③ 理论解法

梯度向量的 `shape` 必须和参数向量完全一样，
才能逐元素对应地更新：

```text
wᵢ ← wᵢ - lr × (∂loss/∂wᵢ)
```

#### ④ 代码实现

```python
y_pred = torch.dot(w, x) + b
loss = (y_pred - 85.0) ** 2
loss.backward()

print(w.grad.shape)   # torch.Size([3])，和 w 一样
print(w.grad)         # 三个梯度值，分别对应三个权重
```

#### ⑤ 手动验证

已知：

```text
loss = (w·x + b - y)²
```

所以：

```text
∂loss/∂w1 = 2(w·x + b - y) × x1
∂loss/∂w2 = 2(w·x + b - y) × x2
∂loss/∂w3 = 2(w·x + b - y) × x3
```

代码验证：

```python
error = (torch.dot(w, x) + b - 85.0).item()

manual_grad_w1 = 2 * error * x[0].item()
manual_grad_w2 = 2 * error * x[1].item()
manual_grad_w3 = 2 * error * x[2].item()

print(f"手算 ∂loss/∂w1: {manual_grad_w1:.4f}")
print(f"代码 w.grad[0]: {w.grad[0].item():.4f}")   # ✓
```

#### ⑥ 破坏实验

```python
# w.grad 的 shape 会不会随着 x 的维度变化？
x_big = torch.randn(10)
w_big = torch.randn(10, requires_grad=True)

loss_big = (torch.dot(w_big, x_big)) ** 2
loss_big.backward()

print(w_big.grad.shape)   # [10]，始终和 w 一样
```

提问：这说明了梯度的什么性质？

### 代码行 4：`for` 循环 vs 无法处理多样本（引出矩阵的必要性）

#### ① 理论问题

现在只能处理**一个样本**。
真实训练有成千上万个样本，怎么同时处理？

#### ② 理论困境

用 `for` 循环逐个处理：

```python
preds = []
for i in range(len(X)):
    pred = torch.dot(w, X[i]) + b
    preds.append(pred)
```

功能上可以，但有两个问题：

* 速度极慢（无法利用 GPU 并行）
* 每个 `pred` 独立，无法批量求 `loss` 再 `backward`

#### ③ 理论解法

把所有样本打包成矩阵 `X`，
用矩阵乘法一次性完成所有点积——
这就是下一个阶段的核心。

#### ④ 代码实现

```python
# 先感受 for 循环的局限
X = torch.tensor([
    [8.0, 7.0, 0.9],
    [5.0, 6.0, 0.7],
    [3.0, 5.0, 0.5],
])

# for 循环方式
preds_loop = []
for i in range(X.shape[0]):
    pred = torch.dot(w.detach(), X[i]) + b.detach()
    preds_loop.append(pred.item())

print(preds_loop)
```

#### ⑤ 手动验证

手算第一个样本：

```text
w · X[0] = 0.6×8 + 0.3×7 + 0.1×0.9 = 6.99
```

```python
print(f"手算: {0.6*8 + 0.3*7 + 0.1*0.9:.2f}")
print(f"循环: {preds_loop[0]:.2f}")   # ✓
```

#### ⑥ 破坏实验

```python
# 用计时器感受速度差异
import time

X_big = torch.randn(10000, 3)
w_det = w.detach()

# for 循环
t0 = time.time()
for i in range(X_big.shape[0]):
    torch.dot(w_det, X_big[i])
print(f"for 循环: {time.time() - t0:.3f}s")

# 矩阵乘法（下一阶段的方式）
t0 = time.time()
X_big @ w_det
print(f"矩阵乘法: {time.time() - t0:.3f}s")
```

差距会让学生直观感受到矩阵运算的必要性。
