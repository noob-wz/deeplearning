## 关键代码行逐一讲解

---

### 代码行 1：`w` 从向量 `[F]` 变成矩阵 `W [F, N_out]`

#### ① 理论问题

上一阶段 `w` 是 `[3]`，输出是 `[样本数]` 的向量——
它只能做一件事：预测一个值。

如果想让网络有多个“中间神经元”同时工作，
每个神经元输出自己的结果，`W` 需要是什么 shape？

#### ② 理论困境

一个向量 `w` 只能产生一列输出。
如果想让 4 个神经元同时工作，
就需要 4 组权重，每组 3 个（对应 3 个输入特征）。

#### ③ 理论解法

把这 4 组权重排成一个矩阵：

* `W` 的 shape = `[3, 4]`
* 3 表示输入特征数
* 4 表示神经元数

并且：

* `W` 的第 `j` 列 = 第 `j` 个神经元的 3 个权重
* `X @ W` 的结果 shape = `[样本数, 4]`

也就是说：

> 每个样本会同时得到 4 个输出，对应 4 个神经元。

#### ④ 代码实现

```python
import torch

X = torch.randn(4, 3)   # 4 个样本，3 个特征

# 一个神经元（上一阶段）
w_one = torch.randn(3, 1)
out_one = X @ w_one
print(f"1个神经元: {out_one.shape}")   # [4, 1]

# 四个神经元（本阶段）
W_four = torch.randn(3, 4)
out_four = X @ W_four
print(f"4个神经元: {out_four.shape}")  # [4, 4]
```

#### ⑤ 手动验证

验证“`W` 的每一列是一个神经元”：

```python
# 第一个神经元的输出 = X @ W[:, 0]
neuron1_out = X @ W_four[:, 0]
print(f"神经元1输出 shape: {neuron1_out.shape}")  # [4]
print(f"等于 out_four 第1列: {torch.allclose(neuron1_out, out_four[:, 0])}")  # True
```

#### ⑥ 破坏实验

```python
# 神经元数从 4 变成 1，shape 怎么变？
W_one = W_four[:, :1]   # 取第一列，shape [3, 1]
out = X @ W_one
print(out.shape)        # [4, 1]

# 神经元数从 4 变成 8，shape 怎么变？
W_eight = torch.randn(3, 8)
out8 = X @ W_eight
print(out8.shape)       # [4, 8]
```

引导学生总结规律：

> 输出 shape 的第二个维度 = 神经元数量

---

### 代码行 2：`W` 的每一列是一个神经元（核心理解）

#### ① 理论问题

“神经元”到底是什么？
它和权重矩阵 `W` 的关系是什么？

#### ② 理论困境

“神经元”这个词听起来很神秘，
但它在数学和代码层面到底对应什么？

#### ③ 理论解法

一个神经元，本质上就是：

> 对输入做一次加权求和，得到一个数

也就是：

```text
torch.dot(w_j, x)
```

其中：

* `x` 是一个样本
* `w_j` 是第 `j` 个神经元的权重

所以：

* `W` 的第 `j` 列 = 第 `j` 个神经元的全部权重
* `X @ W[:, j]` = 第 `j` 个神经元对所有样本的输出
* `X @ W` = 所有神经元对所有样本的输出

#### ④ 代码实现

```python
# 拆开看每个神经元
for j in range(4):
    neuron_j_weights = W_four[:, j]          # shape [3]
    neuron_j_output = X @ neuron_j_weights   # shape [4]
    print(
        f"神经元{j+1} 权重shape: {neuron_j_weights.shape}, "
        f"输出shape: {neuron_j_output.shape}"
    )

# 合在一起
all_neurons_output = X @ W_four   # shape [4, 4]
# 每一列 = 一个神经元的输出
```

#### ⑤ 手动验证

手算第一个样本通过第一个神经元：

```python
x0 = X[0]                 # shape [3]
w0 = W_four[:, 0]         # shape [3]，第一个神经元

manual = (x0 * w0).sum().item()
code = out_four[0, 0].item()

print(f"手算: {manual:.4f}, 代码: {code:.4f}")   # ✓
```

#### ⑥ 破坏实验

```python
# 如果把 W 的所有列初始化成完全一样的值
W_same = torch.ones(3, 4) * 0.5
out_same = X @ W_same
print(out_same)
```

你会看到 4 列完全一样——
也就是 4 个神经元输出完全一样。

引导学生思考：

* 这意味着什么？
* 训练时会怎样？

这里可以自然引出：

> 为什么参数必须随机初始化

### 代码行 3：`nn.Linear` 和手写 `XW + b` 的对应关系

#### ① 理论问题

手写 `X @ W + b` 需要自己管理：

* `W` 和 `b` 的初始化
* `requires_grad`
* 参数更新
* 梯度清零

参数一多，马上就会很乱。

#### ② 理论困境

比如一个两层网络，至少就要手动管理：

* `W1`
* `b1`
* `W2`
* `b2`

再加上更新代码，就会非常混乱。

#### ③ 理论解法

`nn.Linear` 把这一切封装起来了：

* 自动初始化 `W` 和 `b`
* 自动设置 `requires_grad=True`
* 自动管理参数（通过 `.parameters()`）
* 内部执行的是：

```text
input @ W.T + b
```

这里有一个关键点：

> `nn.Linear` 的 `weight` shape 是 `[out_features, in_features]`

也就是：

```text
[神经元数, 特征数]
```

所以 `nn.Linear(3, 4)` 的 `weight.shape` 是 `[4, 3]`，不是 `[3, 4]`。

#### ④ 代码实现

```python
import torch.nn as nn

layer = nn.Linear(in_features=3, out_features=4)

# 验证 weight 的 shape
print(layer.weight.shape)   # [4, 3]
print(layer.bias.shape)     # [4]，每个神经元一个 bias

# 前向传播
out = layer(X)
print(out.shape)            # [4, 4]
```

#### ⑤ 手动验证

```python
# nn.Linear 做的是 X @ W.T + b
manual_out = X @ layer.weight.T + layer.bias
code_out = layer(X)

print(torch.allclose(manual_out, code_out))   # True ✓
```

为什么 `weight` 要转置？

```text
weight.shape   = [4, 3]
weight.T.shape = [3, 4]
X.shape        = [4, 3]

[4, 3] @ [3, 4] → [4, 4]
```

所以这只是 PyTorch 的约定：

> `weight` 按 `[out, in]` 存储，前向时再转置使用。

#### ⑥ 破坏实验

```python
# 如果直接用 weight（不转置）
try:
    wrong_out = X @ layer.weight
except RuntimeError as e:
    print(e)
```

让学生解释：

* 为什么 `[4, 3] @ [4, 3]` 不行？
* 矩阵乘法的 shape 规则是什么？

可以顺便强调：

```text
[m, n] @ [n, k] → [m, k]
```

### 代码行 4：`bias` 的广播（shape `[4]` 加到 shape `[4, 4]` 上）

#### ① 理论问题

`bias` 的 shape 是 `[4]`，表示每个神经元一个 bias。
但 `X @ W` 的输出是 `[4, 4]`，表示：

* 4 个样本
* 4 个神经元输出

shape 不一样，为什么还能相加？

#### ② 理论困境

直觉上，`b[j]` 应该加到第 `j` 列的每个元素上。
也就是说：

> 所有样本经过第 `j` 个神经元时，都加同一个偏置

但 `[4]` 和 `[4, 4]` 明明 shape 不同，直接加不会报错吗？

#### ③ 理论解法

这是 PyTorch 的**广播机制（broadcasting）**：

* `[4]` 会先看成 `[1, 4]`
* 然后自动扩展成 `[4, 4]`
* 再逐元素相加

等价于：

> 把 `b` 复制成 4 行，再和输出相加

#### ④ 代码实现

```python
W = torch.randn(3, 4)
b = torch.randn(4)       # shape [4]
out = X @ W              # shape [4, 4]

result = out + b         # 广播：b 被自动扩展
print(result.shape)      # [4, 4]
```

#### ⑤ 手动验证

```python
# 手动广播验证
b_expanded = b.unsqueeze(0).expand(4, 4)   # [4, 4]
manual_result = out + b_expanded

print(torch.allclose(result, manual_result))   # True ✓
```

再验证：每一行加的 bias 是一样的吗？

```python
print(result[0] - result[1])
print((out[0] + b) - (out[1] + b))   # 差值不含 b
```

#### ⑥ 破坏实验

```python
# b 的 shape 改成 [3] 会怎样？
b_wrong = torch.randn(3)

try:
    out + b_wrong
except RuntimeError as e:
    print(e)
```

因为：

```text
[4, 4] + [3]
```

最后一个维度 `4 ≠ 3`，无法广播。

## 完整代码：第一个真正的神经网络层

```python
import torch
import torch.nn as nn

# 数据
X = torch.randn(32, 3)   # 32 个样本，3 个特征
y = torch.randn(32, 1)   # 32 个目标值

# 模型：一个线性层，3 个输入，4 个神经元，输出 1 个值
# 这里用两层来引出下一阶段
layer1 = nn.Linear(3, 4)    # 3 → 4 个神经元
layer2 = nn.Linear(4, 1)    # 4 → 1 个输出

optimizer = torch.optim.SGD(
    list(layer1.parameters()) + list(layer2.parameters()),
    lr=0.01
)

for epoch in range(200):
    # 前向传播（注意：两层线性，中间没有激活函数）
    hidden = layer1(X)    # [32, 3] → [32, 4]
    out = layer2(hidden)  # [32, 4] → [32, 1]

    loss = ((out - y) ** 2).mean()

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if epoch % 40 == 0:
        print(f"epoch {epoch}: loss={loss.item():.4f}")
```

让学生跑完后回答：

> 两层 `nn.Linear` 叠在一起，效果比一层好吗？
> 你来实验——用一层 `nn.Linear(3, 1)` 对比。
> 结果让你意外吗？为什么？

这个实验会自然引出下一阶段的核心问题。
