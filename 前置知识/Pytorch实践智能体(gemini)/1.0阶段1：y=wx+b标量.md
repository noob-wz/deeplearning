# 阶段 01：`y = wx + b`，最简单的形式

## 在主线的位置

`[标量] → 向量 → 矩阵 → 多神经元 → 多层网络`

## 核心问题

什么叫预测？机器学习在优化什么？
“学习”最简单的形式是什么？

---

## 认知冲突触发

不讲任何理论，直接抛出情境：

> 一个人每天学习 2 小时，考了 70 分。
> 另一个人每天学习 5 小时，考了 85 分。
> 现在有人每天学习 8 小时，你猜他能考多少分？
> 你用什么方法得出这个答案的？

让学生回答，再问：

> 你脑子里有没有一条线？
> 这条线怎么用一个式子写出来？

自然引出：`y = wx + b`

---

## 直觉建立

`y = wx + b` 就是一条直线：

* `x` → 你知道的（输入：学习时间）
* `y` → 你想预测的（输出：考试分数）
* `w` → 斜率（每多学 1 小时，分数大概涨多少）
* `b` → 截距（学习时间为 0 时的基础分）

训练的目标：

> 找到最好的 `w` 和 `b`，让这条直线尽量穿过所有数据点。

---

## 关键代码行逐一讲解

---

### 代码行 1：`requires_grad=True`

#### ① 理论问题

我们需要找到最好的 `w` 和 `b`。
“最好”意味着要调整 `w` 和 `b`——
但怎么知道往哪个方向调？

需要知道：

* `w` 变化时，`loss` 怎么变化

数学上这就是：

`∂loss/∂w`，即 `loss` 对 `w` 的偏导数。

#### ② 理论困境

`w` 和 `b` 是我们定义的普通数字，PyTorch 默认不会追踪对它们的任何运算，也就无从计算偏导数。

#### ③ 理论解法

需要告诉 PyTorch：

> 这个变量是需要被优化的参数，
> 请记录所有涉及它的运算，以便之后求导。

#### ④ 代码实现

```python
w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)
```

`requires_grad=True` 的作用：

> PyTorch 开始构建“计算图”——
> 记录 `w` 参与了哪些运算，以便之后反向传播。

#### ⑤ 手动验证

```python
# 验证追踪是否开启
print(w.requires_grad)   # True
print(w.grad)            # None ← 还没有计算梯度

# 对比：普通 tensor 不追踪
w_normal = torch.tensor(1.0)
print(w_normal.requires_grad)   # False
```

#### ⑥ 破坏实验

```python
# 去掉 requires_grad，看之后 backward() 会怎样
w_bad = torch.tensor(1.0)
loss = (w_bad * 2.0 - 3.0) ** 2
loss.backward()   # 报错：没有需要求导的叶子节点
```

让学生读报错信息，解释为什么。

---

### 代码行 2：`y_pred = w * x + b`（前向传播）

#### ① 理论问题

我们有了参数 `w` 和 `b`，有了输入 `x`，
需要计算模型的预测值。

这在理论上叫“前向传播”——
信息从输入流向输出。

#### ② 理论困境

没有困境，这就是直接套公式。
但关键是：这步运算必须被计算图记录，
才能在之后正确求导。

#### ③ 理论解法

```text
y_pred = w × x + b
```

这就是线性函数，参数是 `w` 和 `b`。

#### ④ 代码实现

```python
x = torch.tensor([2.0, 5.0, 1.0, 8.0, 3.0])
y = torch.tensor([70.0, 85.0, 60.0, 95.0, 72.0])

y_pred = w * x + b
print(y_pred)
# tensor([2., 5., 1., 8., 3.], grad_fn=<AddBackward0>)
```

注意 `grad_fn=<AddBackward0>`：

这说明 PyTorch 正在追踪这个运算，计算图已建立。

#### ⑤ 手动验证

当前 `w = 1`，`b = 0`，手算第一个样本：

```text
y_pred[0] = 1.0 × 2.0 + 0.0 = 2.0
真实值 y[0] = 70.0
差了 68 分——说明现在的参数很差
```

```python
print(y_pred[0].item())   # 2.0 ✓ 和手算一致
```

#### ⑥ 破坏实验

```python
# 如果 w 和 b 没有 requires_grad，
# y_pred 的 grad_fn 会是什么？
w2 = torch.tensor(1.0)
b2 = torch.tensor(0.0)
y_pred2 = w2 * x + b2
print(y_pred2.grad_fn)   # None ← 没有被追踪
```

让学生解释：这意味着什么？之后还能求导吗？

---

### 代码行 3：`loss = ((y_pred - y) ** 2).mean()`（MSE 损失）

#### ① 理论问题

我们需要一个数字来衡量“模型现在有多差”。
这个数字需要满足：

* 预测越准，数字越小（最好是 0）
* 数字对 `w` 和 `b` 是可以求导的
* 对大误差的惩罚要比小误差更重

#### ② 理论困境

最直觉的想法是直接用误差：

```python
error = y_pred - y
```

但 `error` 是正负混合的，加起来会互相抵消——
模型可能学出“平均正确但每个都错”的结果。

用绝对值呢？

`|error|` 在 `error = 0` 时不可导，梯度在此无定义。

#### ③ 理论解法

用平方：

```text
(y_pred - y)²
```

优点：

* 始终为正，不会互相抵消
* 处处可导
* 大误差惩罚更重（`68² >> 2²`）

再用 `.mean()` 对所有样本取平均：

* `loss` 的大小与样本数无关
* 换 batch size 不需要重新调学习率

#### ④ 代码实现

```python
loss = ((y_pred - y) ** 2).mean()
print(f"当前 loss: {loss.item():.2f}")
# 很大，因为参数还是随机的
```

#### ⑤ 手动验证

只用第一个样本手算：

```text
y_pred[0] = 2.0, y[0] = 70.0
误差 = 2.0 - 70.0 = -68.0
平方 = (-68.0)² = 4624.0
5 个样本平均 = 各自平方和 / 5
```

```python
manual_errors = (y_pred - y) ** 2
print(manual_errors)          # 看每个样本的平方误差
print(manual_errors.mean())   # 和 loss 对比，应该一样
```

#### ⑥ 破坏实验

```python
# 实验 1：去掉平方，用原始误差
loss_bad1 = (y_pred - y).mean()
print(f"不平方的 loss: {loss_bad1.item():.2f}")
# 接近 0？但模型明显很差——解释为什么

# 实验 2：去掉 .mean()
loss_bad2 = (y_pred - y) ** 2
print(loss_bad2.shape)   # [5]，不是标量
loss_bad2.backward()     # 报错！backward 需要标量 loss
# 让学生读报错，解释为什么 loss 必须是标量
```

### 代码行 4：`loss.backward()`（反向传播）

#### ① 理论问题

我们有了 `loss`，需要知道：

* 如果把 `w` 稍微调大一点，`loss` 会变大还是变小？
* 会变多少？

数学上这就是 `∂loss/∂w`。

#### ② 理论困境

`loss` 是 `y_pred` 的函数，`y_pred` 是 `w` 的函数。
要求 `∂loss/∂w`，需要用链式法则：

```text
∂loss/∂w = (∂loss/∂y_pred) × (∂y_pred/∂w)
```

模型越复杂，链条越长，手算不可能。

#### ③ 理论解法

PyTorch 在前向传播时已经记录了计算图。
`backward()` 沿着这个图从 `loss` 出发，
用链式法则自动计算每个参数的偏导数，
存入对应参数的 `.grad` 属性。

#### ④ 代码实现

```python
loss.backward()
print(w.grad)   # ∂loss/∂w
print(b.grad)   # ∂loss/∂b
```

#### ⑤ 手动验证

用最简单的情况手算验证：

```python
# 只有一个样本，x=2, y=3
w_test = torch.tensor(1.0, requires_grad=True)
b_test = torch.tensor(0.0, requires_grad=True)

y_pred_test = w_test * 2.0 + b_test   # = 2.0
loss_test = (y_pred_test - 3.0) ** 2  # = (2-3)² = 1.0

loss_test.backward()

# 手算：
# loss = (wx + b - y)²
# ∂loss/∂w = 2(wx + b - y) × x
#           = 2 × (1×2+0-3) × 2
#           = 2 × (-1) × 2 = -4.0
print(w_test.grad)   # 应该是 -4.0，验证手算 ✓

# ∂loss/∂b = 2(wx + b - y) × 1
#           = 2 × (-1) × 1 = -2.0
print(b_test.grad)   # 应该是 -2.0 ✓
```

#### ⑥ 破坏实验

```python
# 实验 1：backward() 之前 .grad 是什么？
w2 = torch.tensor(1.0, requires_grad=True)
print(w2.grad)            # None
loss2 = (w2 * 2 - 3) ** 2
# 不调用 backward()
print(w2.grad)            # 还是 None

# 实验 2：连续两次 backward() 不清零
loss2.backward()
print(w2.grad)            # -4.0
loss2 = (w2 * 2 - 3) ** 2
loss2.backward()
print(w2.grad)            # -8.0！累加了
```

让学生解释为什么，引出下一个代码行。

### 代码行 5：`with torch.no_grad(): w -= lr * w.grad`（参数更新）

#### ① 理论问题

我们有了梯度 `∂loss/∂w`，
现在需要用梯度来更新参数。

* 方向：梯度的反方向（梯度指向上坡，我们要下坡）
* 步长：由学习率 `lr` 控制

#### ② 理论困境

如果直接写：

```python
w = w - lr * w.grad
```

PyTorch 会把这个更新操作也纳入计算图——
下次 `backward()` 时会错误地对更新操作求导。

#### ③ 理论解法

参数更新不是“计算过程”，而是“改变参数值”，
不需要被计算图追踪。
用 `torch.no_grad()` 临时关闭追踪。

#### ④ 代码实现

```python
lr = 0.001
with torch.no_grad():
    w -= lr * w.grad   # w = w - lr × ∂loss/∂w
    b -= lr * b.grad   # b = b - lr × ∂loss/∂b
```

#### ⑤ 手动验证

更新前：

```text
w = 1.0
w.grad = -4.0
lr = 0.001
```

更新后：

```text
w = 1.0 - 0.001 × (-4.0) = 1.004
```

梯度是负的，说明 `w` 应该增大，`loss` 才会减小。

验证更新之后 `loss` 是否变小：

```python
y_pred_new = w * x + b
loss_new = ((y_pred_new - y) ** 2).mean()
print(f"更新前 loss: {loss.item():.4f}")
print(f"更新后 loss: {loss_new.item():.4f}")   # 应该变小
```

#### ⑥ 破坏实验

```python
# 实验 1：去掉 torch.no_grad()
w3 = torch.tensor(1.0, requires_grad=True)
w3 = w3 - 0.001 * w3.grad
# w3 现在有 grad_fn，下次 backward() 行为异常

# 实验 2：学习率太大
lr_bad = 10.0
with torch.no_grad():
    w_bad = w - lr_bad * w.grad
y_pred_bad = w_bad * x + b
loss_bad = ((y_pred_bad - y) ** 2).mean()
print(f"lr太大时 loss: {loss_bad.item():.2f}")
# loss 反而更大！让学生解释为什么
```

### 代码行 6：`w.grad.zero_()`（梯度清零）

#### ① 理论问题

梯度下降的每一步需要的是：

> 基于当前参数位置的梯度

而不是过去所有步梯度的累积。

#### ② 理论困境

PyTorch 默认把每次 `backward()` 的梯度
累加到 `.grad` 上，不会自动清零。

这是一个设计选择（某些场景下累加有用），
但普通训练中必须手动清零。

#### ③ 理论解法

每次参数更新之后，立刻清零梯度，
为下一次 `backward()` 做准备。

#### ④ 代码实现

```python
w.grad.zero_()
b.grad.zero_()
```

注意下划线结尾：原地操作（in-place），
不创建新 tensor，直接修改现有 tensor。

#### ⑤ 手动验证

```python
# 验证清零之后 .grad 变成了什么
print(w.grad)      # 某个非零值
w.grad.zero_()
print(w.grad)      # tensor(0.)
```

#### ⑥ 破坏实验

```python
# 去掉 zero_()，跑 5 步，观察梯度变化
w_test = torch.tensor(3.0, requires_grad=True)
for i in range(5):
    loss_t = w_test ** 2
    loss_t.backward()
    print(f"第{i+1}步 grad: {w_test.grad.item():.2f}")
    with torch.no_grad():
        w_test -= 0.1 * w_test.grad
    # 故意不清零

# 对比：有 zero_() 的版本
w_test2 = torch.tensor(3.0, requires_grad=True)
for i in range(5):
    loss_t2 = w_test2 ** 2
    loss_t2.backward()
    print(f"第{i+1}步 grad: {w_test2.grad.item():.2f}")
    with torch.no_grad():
        w_test2 -= 0.1 * w_test2.grad
    w_test2.grad.zero_()
```

让学生对比两个版本的梯度值和收敛行为，
解释为什么不清零会导致训练不稳定。

## 完整训练循环（把六行代码串起来）

```python
import torch

# 数据
x = torch.tensor([2.0, 5.0, 1.0, 8.0, 3.0])
y = torch.tensor([70.0, 85.0, 60.0, 95.0, 72.0])

# 参数初始化
w = torch.tensor(1.0, requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)
lr = 0.001

for epoch in range(300):
    # ① 前向传播：计算预测值
    y_pred = w * x + b

    # ② 计算损失：衡量预测有多差
    loss = ((y_pred - y) ** 2).mean()

    # ③ 反向传播：计算每个参数的梯度
    loss.backward()

    # ④ 参数更新：沿梯度反方向走一步
    with torch.no_grad():
        w -= lr * w.grad
        b -= lr * b.grad

    # ⑤ 梯度清零：为下一次 backward 准备
    w.grad.zero_()
    b.grad.zero_()

    if epoch % 60 == 0:
        print(f"epoch {epoch:3d}: "
              f"loss={loss.item():.2f}, "
              f"w={w.item():.4f}, "
              f"b={b.item():.4f}")
```

让学生在跑通之后回答：

> 这六步的顺序能不能换？
> 比如把 `zero_()` 放到 `backward()` 之前？
> 把 `backward()` 放到 `step` 之后？
> 每种换法会发生什么？
