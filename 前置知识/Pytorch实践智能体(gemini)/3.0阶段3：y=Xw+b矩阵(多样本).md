### 代码行 1：`X` 从向量变成矩阵，shape 是 `[样本数, 特征数]`

#### ① 理论问题

多个样本，每个样本有多个特征，
需要一个数学结构同时表示所有样本。

#### ② 理论困境

一个向量只能表示：

* 一个样本的所有特征，或者
* 所有样本的某一个特征

它无法同时表示“多个样本 × 多个特征”。

#### ③ 理论解法

用**矩阵**表示：

* 行 = 样本
* 列 = 特征

所以：

* `X` 的 shape = `[样本数 N, 特征数 F]`
* `X[i]` = 第 `i` 个样本的所有特征（一行）
* `X[:, j]` = 所有样本的第 `j` 个特征（一列）

#### ④ 代码实现

```python
import torch

X = torch.tensor([
    [8.0, 7.0, 0.9],   # 样本 0
    [5.0, 6.0, 0.7],   # 样本 1
    [3.0, 5.0, 0.5],   # 样本 2
    [9.0, 8.0, 1.0],   # 样本 3
])   # shape: [4, 3]

print(X.shape)      # torch.Size([4, 3])
print(X[0])         # 第一个样本：tensor([8., 7., 0.9])
print(X[:, 0])      # 所有样本的学习时间特征
```

#### ⑤ 手动验证

```python
# shape 的含义
N, F = X.shape
print(f"样本数: {N}, 特征数: {F}")

# 第二个样本的第三个特征
print(X[1, 2].item())   # 0.7，手动确认 ✓
```

#### ⑥ 破坏实验

```python
# 如果行列搞反，X shape 是 [3, 4] 而不是 [4, 3]？
X_wrong = X.T   # 转置
print(X_wrong.shape)   # [3, 4]
```

提问学生：

* 现在 `X_wrong[0]` 代表什么？
* 它还是“第一个样本”吗？

### 代码行 2：`X @ w`（矩阵向量乘法）

#### ① 理论问题

`X` 是 `[4, 3]` 的矩阵，`w` 是 `[3]` 的向量，
需要同时对 4 个样本做点积，得到 4 个预测值。

#### ② 理论困境

`torch.dot()` 只支持两个 1D 向量，
不能直接用于矩阵和向量的运算。

#### ③ 理论解法

用矩阵向量乘法：

```text
Y = X @ w
```

它等价于：对 `X` 的每一行分别和 `w` 做点积。

```text
Y[0] = X[0] · w
Y[1] = X[1] · w
Y[2] = X[2] · w
Y[3] = X[3] · w
```

所以：

* 结果 `Y` 的 shape = `[4]`
* 表示 4 个样本各自的预测值

shape 规则是：

```text
[4, 3] @ [3] → [4]
```

前一个矩阵的列数，必须等于后一个向量的长度。

#### ④ 代码实现

```python
w = torch.tensor([0.6, 0.3, 0.1], requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)

Y_pred = X @ w + b
print(Y_pred.shape)   # torch.Size([4])
print(Y_pred)         # 4 个预测值
```

#### ⑤ 手动验证

手算第一个样本：

```text
X[0] · w
= 0.6×8.0 + 0.3×7.0 + 0.1×0.9
= 4.8 + 2.1 + 0.09
= 6.99
```

```python
manual_0 = (X[0] * w.detach()).sum()
print(f"手算 Y[0]: {manual_0.item():.2f}")
print(f"矩阵乘法 Y[0]: {Y_pred[0].item():.2f}")   # ✓
```

验证它等价于逐行点积：

```python
for i in range(4):
    manual_i = torch.dot(w.detach(), X[i])
    print(f"样本{i}: 手动={manual_i.item():.2f}, 矩阵={Y_pred[i].item():.2f}")
```

#### ⑥ 破坏实验

```python
# shape 不匹配会怎样？
w_wrong = torch.tensor([0.6, 0.3], requires_grad=True)

try:
    X @ w_wrong
except RuntimeError as e:
    print(e)
```

让学生解释：为什么 `[4, 3] @ [2]` 不行？

再看另一种情况：

```python
# 注意 w 的 shape：[3] vs [3, 1] 结果有何不同？
w_col = w.detach().unsqueeze(1)   # [3, 1]
print((X @ w_col).shape)          # [4, 1]，多了一个维度
```

提问学生：

* `[4]` 和 `[4, 1]` 有什么区别？
* 哪个更常用？

### 代码行 3：`loss = ((Y_pred - y) ** 2).mean()`（多样本 loss）

#### ① 理论问题

现在 `Y_pred` 是 4 个预测值，`y` 是 4 个真实值。
需要把 4 个样本的误差合并成一个**标量** `loss`，
才能用 `backward()` 求梯度。

#### ② 理论困境

`(Y_pred - y)` 是一个 `[4]` 的向量，不是标量。
而 `backward()` 默认需要的是标量 loss，
否则无法确定“对哪个输出求导”。

#### ③ 理论解法

对每个样本的平方误差取平均：

```text
L = (1/N) × Σᵢ (ŷᵢ - yᵢ)²
```

这样得到的结果是一个标量，表示：

> 平均每个样本的误差有多大

取平均而不是求和的好处：

* `loss` 的大小与样本数无关
* 换 batch size 时，不必明显重调学习率

#### ④ 代码实现

```python
y = torch.tensor([85.0, 78.0, 65.0, 92.0])

errors = Y_pred - y          # [4]，每个样本的误差
sq_errors = errors ** 2      # [4]，平方误差
loss = sq_errors.mean()      # 标量

print(f"loss shape: {loss.shape}")   # torch.Size([])
print(f"loss: {loss.item():.4f}")
```

#### ⑤ 手动验证

```python
# 手算：平方误差之和 / 4
manual_errors = [(Y_pred[i].item() - y[i].item()) ** 2 for i in range(4)]
manual_loss = sum(manual_errors) / 4

print(f"手算 loss: {manual_loss:.4f}")
print(f"代码 loss: {loss.item():.4f}")   # ✓
```

#### ⑥ 破坏实验

```python
# 用 .sum() 代替 .mean() 会怎样？
loss_sum = ((Y_pred - y) ** 2).sum()
print(f"sum loss: {loss_sum.item():.4f}")
# 是 mean 的 4 倍
```

提问学生：

* 如果 batch size 从 4 变成 8，`sum loss` 会怎样？
* `mean loss` 又会怎样？

让学生先推理，再用代码验证。

### 代码行 4：`w.grad` 的 shape 依然是 `[3]`（不随样本数变化）

#### ① 理论问题

有 4 个样本，每个样本都贡献了对 `w` 的梯度。
那最终 `w.grad` 的 shape 会是什么？

#### ② 理论困境

既然有 4 个样本、每个样本对 3 个权重都有影响，
会不会 `w.grad` 变成 `[4, 3]`？

#### ③ 理论解法

不会。

因为我们求的是：

```text
loss = mean(所有样本的误差)
```

对 `w` 求导时，4 个样本的梯度贡献会被合并，
最终 `w.grad` 仍然是 `[3]`：

* 每个元素对应一个权重
* 每个值表示“这个权重在当前 batch 上的总体梯度”

这就是 mini-batch SGD 的数学基础。

#### ④ 代码实现

```python
loss.backward()
print(w.grad.shape)   # torch.Size([3])，不是 [4, 3]
print(w.grad)         # 3 个梯度值，每个对应一个权重
```

#### ⑤ 手动验证

对于 MSE loss，可以写成：

```text
∂loss/∂w = (2/N) × X.T @ (Y_pred - y)
```

这里重点不是要求学生完全推导，
而是让他们看到手算结果和自动求导结果一致。

```python
manual_grad = (2.0 / 4) * (X.T @ (Y_pred - y).detach())
print(f"手算梯度: {manual_grad}")
print(f"代码梯度: {w.grad}")   # ✓
```

#### ⑥ 破坏实验

```python
# batch size 从 4 变成 2，w.grad 的 shape 会变吗？
X2 = X[:2]
y2 = y[:2]

w2 = torch.tensor([0.6, 0.3, 0.1], requires_grad=True)
loss2 = ((X2 @ w2 - y2) ** 2).mean()
loss2.backward()

print(w2.grad.shape)   # 还是 [3]
```

提问学生：

* 这说明了什么？
* 为什么样本数变了，参数梯度的 shape 却没变？

## 完整训练循环

```python
import torch

X = torch.tensor([
    [8.0, 7.0, 0.9],
    [5.0, 6.0, 0.7],
    [3.0, 5.0, 0.5],
    [9.0, 8.0, 1.0],
])
y = torch.tensor([85.0, 78.0, 65.0, 92.0])

w = torch.randn(3, requires_grad=True)
b = torch.tensor(0.0, requires_grad=True)
lr = 0.001

for epoch in range(300):
    Y_pred = X @ w + b               # 前向传播，shape [4]
    loss = ((Y_pred - y) ** 2).mean()  # 标量 loss

    loss.backward()                  # 梯度：w.grad shape [3]

    with torch.no_grad():
        w -= lr * w.grad
        b -= lr * b.grad

    w.grad.zero_()
    b.grad.zero_()

    if epoch % 60 == 0:
        print(f"epoch {epoch}: loss={loss.item():.4f}")
```

让学生在跑通后回答：

> `w` 最终的值是什么？
> `w[0]` 比 `w[1]` 大还是小？
> 这说明三个特征里，哪个对成绩影响最大？
