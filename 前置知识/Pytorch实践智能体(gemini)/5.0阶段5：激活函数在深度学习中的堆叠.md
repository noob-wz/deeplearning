## 认知冲突触发（代码先行）

> 我们来证明一件事：
> 两层线性 = 一层线性，深度没有意义。

```python
import torch
import torch.nn as nn

layer1 = nn.Linear(3, 4, bias=False)
layer2 = nn.Linear(4, 1, bias=False)

X = torch.randn(5, 3)

# 两层线性
out_two = layer2(layer1(X))

# 等价的单层：把两个权重矩阵合并
W_combined = layer2.weight @ layer1.weight
out_one = X @ W_combined.T

print(torch.allclose(out_two, out_one, atol=1e-5))   # True
```

然后问学生：

> 两层等于一层——这说明什么？
> 不管你堆多少层线性，都可以合并成一层。
> 深度学习的“深”从哪里来？

## 关键代码行逐一讲解

---

### 代码行 1：证明多层线性 = 一层线性

#### ① 理论问题

为什么无论堆多少层线性层，
都可以等价为一层线性层？

#### ② 理论困境

直觉上，更多层应该能做更多事情。
但数学告诉我们不是这样。

#### ③ 理论解法

两层线性可以写成：

```text
第一层：H = X @ W1
第二层：Y = H @ W2
```

合并后：

```text
Y = (X @ W1) @ W2 = X @ (W1 @ W2)
```

而 `W1 @ W2` 还是一个矩阵。
把它记成 `W_combined`，就得到：

> 两层线性 = 一层线性（只是换了一个合并后的权重矩阵）

三层、四层也一样，可以继续递归合并。

结论：

> **没有非线性，深度没有意义。**

#### ④ 代码实现

```python
# 见上方“认知冲突触发”中的代码
print(torch.allclose(out_two, out_one, atol=1e-5))   # True
```

#### ⑤ 手动验证

```python
# 手算 W_combined
W1 = layer1.weight.detach()   # [4, 3]
W2 = layer2.weight.detach()   # [1, 4]
W_combo = W2 @ W1             # [1, 3]

# 用 W_combo 直接预测
out_manual = X @ W_combo.T    # [5, 1]

print(torch.allclose(layer2(layer1(X)), out_manual, atol=1e-5))   # True ✓
```

#### ⑥ 破坏实验

```python
# 在两层之间加一个激活函数之后，还能合并吗？
out_with_relu = layer2(torch.relu(layer1(X)))

# 试着用同样的 W_combined 复现
out_combined = X @ W_combined.T

print(torch.allclose(out_with_relu, out_combined, atol=1e-5))   # False
```

加了激活函数，就无法再合并了。

引导学生思考：

> 为什么？`ReLU` 破坏了什么？

### 代码行 2：`F.relu(x)`（第一个激活函数）

#### ① 理论问题

我们需要在线性变换之间加入“非线性”，
打破“多层 = 一层”的等价性。

这个非线性操作需要尽量满足：

* 简单，计算快
* 处处或几乎处处可导（才能 `backward`）
* 能真正引入非线性

#### ② 理论困境

最直觉的非线性也许是 `x²`，
但梯度会随层数迅速放大。

`Sigmoid` 曾经很流行，
但在深层网络中又容易出现梯度消失。

#### ③ 理论解法

最经典的选择之一是 **ReLU（Rectified Linear Unit）**：

```text
f(x) = max(0, x)
```

也就是：

* 负数变成 0
* 正数保持不变

直觉上可以理解成：

> 把负数那一半“折叠”到 0 上

这一步引入了真正的弯折，
因此已经不能再被单层线性函数复现。

它的导数近似是：

* `x > 0` 时，导数为 `1`
* `x < 0` 时，导数为 `0`
* `x = 0` 处通常按工程实现取 `0`

所以它有几个重要优点：

* 几乎处处可导
* 正数区域梯度不衰减
* 计算非常便宜

#### ④ 代码实现

```python
import torch
import torch.nn.functional as F

x = torch.tensor([-3.0, -1.0, 0.0, 1.0, 3.0])
out = F.relu(x)
print(out)   # tensor([0., 0., 0., 1., 3.])
```

#### ⑤ 手动验证

```python
# 手算每个元素：max(0, x)
manual = [max(0, v) for v in [-3, -1, 0, 1, 3]]
print(manual)   # [0, 0, 0, 1, 3] ✓

# 对比 Sigmoid（用于感受差异）
sig = torch.sigmoid(x)
print(sig)   # 所有值都在 (0,1) 之间，压缩更严重
```

#### ⑥ 破坏实验

```python
# ReLU 的梯度在负数区域是什么？
x_neg = torch.tensor([-2.0], requires_grad=True)
out_neg = F.relu(x_neg)
out_neg.backward()
print(x_neg.grad)   # 0.0
```

这意味着：

> 如果一个神经元的输入长期为负，它的梯度可能一直是 0，参数很难更新。

这就是常说的“死亡神经元”现象。

可以让学生继续思考：

* 这会成为实际问题吗？
* 为什么还会有人继续使用 ReLU？

### 代码行 3：`nn.Module` 和 `forward()`（封装网络）

#### ① 理论问题

网络一旦有多层，就需要一种组织方式来做到：

* 把所有层放在一起
* 清楚定义数据从输入到输出的流动路径
* 让 PyTorch 自动找到所有参数

#### ② 理论困境

直接把 `layer1`、`layer2` 散着写，
代码没有结构，参数管理也很混乱，难以复用。

#### ③ 理论解法

`nn.Module` 提供了标准封装方式：

* `__init__()`：定义并注册所有层
* `forward()`：定义数据如何流动

这样做之后：

* 参数会自动注册
* `model.parameters()` 会自动收集所有可训练参数
* 网络结构更清晰，也更容易扩展

#### ④ 代码实现

```python
import torch.nn as nn
import torch.nn.functional as F

class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        # 在这里注册层，PyTorch 会自动追踪它们的参数
        self.layer1 = nn.Linear(3, 4)
        self.layer2 = nn.Linear(4, 1)

    def forward(self, x):
        # 数据流动路径
        x = self.layer1(x)    # 线性变换，[batch, 4]
        x = F.relu(x)         # 非线性，[batch, 4]
        x = self.layer2(x)    # 线性变换，[batch, 1]
        return x

model = SimpleNet()
```

#### ⑤ 手动验证

```python
# 验证 model.parameters() 包含所有参数
total_params = sum(p.numel() for p in model.parameters())
print(f"总参数数: {total_params}")
# layer1: 3×4 + 4 = 16
# layer2: 4×1 + 1 = 5
# 合计: 21 ✓
```

再验证 `forward()` 的 shape 流动：

```python
X_test = torch.randn(8, 3)    # 8 个样本
out = model(X_test)
print(out.shape)              # [8, 1] ✓
```

这里很适合要求学生：

> 逐行标注 `forward()` 中每一步的 shape

#### ⑥ 破坏实验

```python
# 在 forward 中去掉 relu 会怎样？
class LinearOnlyNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(3, 4)
        self.layer2 = nn.Linear(4, 1)

    def forward(self, x):
        x = self.layer1(x)
        x = self.layer2(x)   # 没有 relu
        return x
```

然后引导学生设计实验：

* 在同一个任务上训练 `SimpleNet`
* 再训练 `LinearOnlyNet`
* 对比两者的 loss 曲线和拟合能力

### 代码行 4：`optimizer = torch.optim.SGD / Adam`

#### ① 理论问题

参数更新的基本公式是：

```text
θ ← θ - lr × ∂loss/∂θ
```

但当参数数量很多时，手动为每个参数写更新代码根本不可扩展。

#### ② 理论困境

`SimpleNet` 只有 21 个参数，
手动写更新还勉强能做。

可一旦参数上万、上百万，
手写更新完全不现实。

而且，基础的 SGD 在某些任务上收敛会比较慢。

#### ③ 理论解法

PyTorch 用 `optimizer` 封装参数更新逻辑。

常见两种：

* `SGD`：最基础的梯度下降
* `Adam`：自适应学习率方法，通常前期收敛更快

它们都能直接接收：

```python
model.parameters()
```

也就是模型里所有需要训练的参数。

#### ④ 代码实现

```python
# SGD：基础梯度下降
optimizer_sgd = torch.optim.SGD(
    model.parameters(),
    lr=0.01
)

# Adam：自适应，通常收敛更快
optimizer_adam = torch.optim.Adam(
    model.parameters(),
    lr=0.001
)
```

#### ⑤ 手动验证

```python
# 验证 optimizer 管理的参数就是 model.parameters()
for p1, p2 in zip(model.parameters(), optimizer_sgd.param_groups[0]['params']):
    print(torch.allclose(p1, p2))   # True ✓
```

你也可以让学生进一步验证：

> `optimizer.step()` 是否等价于“按梯度手动更新参数”

#### ⑥ 破坏实验

这里很适合做对比实验：

* 用 `SGD` 训练同一个任务
* 用 `Adam` 训练同一个任务
* 画出两条 loss 曲线

通常学生会观察到：

> Adam 在前期收敛更快一些

### 代码行 5：训练循环的标准写法（三行核心）

#### ① 理论问题

每次更新参数，都要完成三件事：

1. 清零上一次的梯度
2. 计算本次梯度
3. 更新参数

这三步的顺序能不能调换？

#### ② 理论困境

初学者特别容易：

* 忘记清零梯度
* 把顺序写错
* 导致训练结果异常，却不知道问题出在哪

#### ③ 理论解法

标准顺序是：

```text
zero_grad → backward → step
```

原因非常明确：

* `backward()` 会把梯度**累加**到 `.grad`
* 所以必须先 `zero_grad()`，清掉上一次残留的梯度
* `step()` 依赖 `.grad` 更新参数
* 所以必须等 `backward()` 之后再执行

也就是说：

* `zero_grad()` 为什么在前面？因为要先清干净
* `step()` 为什么在后面？因为要等梯度先算出来

#### ④ 代码实现

```python
for epoch in range(200):
    out = model(X)
    loss = ((out - y) ** 2).mean()

    optimizer.zero_grad()   # ① 清零梯度
    loss.backward()         # ② 计算梯度
    optimizer.step()        # ③ 更新参数

    if epoch % 40 == 0:
        print(f"epoch {epoch}: loss={loss.item():.4f}")
```

#### ⑤ 手动验证

这里可以回扣阶段 01 的内容：

* 以前是手动写 `w.grad.zero_()`
* 现在是 `optimizer.zero_grad()`

本质上是一样的，区别只是：

> `optimizer.zero_grad()` 会对模型里所有参数统一清零

#### ⑥ 破坏实验

可以让学生逐个试错：

* 把 `zero_grad()` 移到 `step()` 之后
* 完全去掉 `zero_grad()`
* 把 `step()` 放到 `backward()` 之前

然后观察：

* loss 是否异常
* 是否出现梯度累积
* 参数更新是否滞后

最后要求他们用语言解释：

> 每种错误为什么会发生问题？

## 完整训练循环（最终版）

```python
import torch
import torch.nn as nn
import torch.nn.functional as F

# 数据
X = torch.randn(100, 3)
y = torch.randn(100, 1)

# 模型
class SimpleNet(nn.Module):
    def __init__(self):
        super().__init__()
        self.layer1 = nn.Linear(3, 4)
        self.layer2 = nn.Linear(4, 1)

    def forward(self, x):
        x = self.layer1(x)   # [batch, 4]
        x = F.relu(x)        # [batch, 4]，非线性
        x = self.layer2(x)   # [batch, 1]
        return x

model = SimpleNet()
optimizer = torch.optim.Adam(model.parameters(), lr=0.001)

# 训练
for epoch in range(300):
    out = model(X)
    loss = ((out - y) ** 2).mean()

    optimizer.zero_grad()
    loss.backward()
    optimizer.step()

    if epoch % 60 == 0:
        print(f"epoch {epoch}: loss={loss.item():.4f}")
```

让学生在跑完后做三件事：

1. 数出模型总参数数，手算验证
2. 在 `forward()` 里每行代码旁边标注当前 shape
3. 把 `ReLU` 去掉，对比两个版本的 loss 曲线，解释差异
