## 关于"简化版冲压成形"的诚实定位

**真实冲压成形仿真包含什么**：
- **大变形**：板料从平面状态拉伸到深凹型腔，变形量通常几倍于板厚
- **非线性本构**：材料进入塑性区域（`σ = σ_y + K·ε^n` 硬化模型或更复杂的屈服面）
- **接触**：凸模-板料-凹模三者之间的接触压力、摩擦、滑动
- **显式动力学求解**：LS-DYNA 风格的时间步进 + 大量迭代
- **各种失效判据**：FLD 曲线、厚度减薄率、起皱预测

**Part G 做的简化版做什么**：
- ✅ 2D 弹塑性变形（比线弹性更进一步，加入塑性硬化）
- ✅ 准静态分析（不做时间步进）
- ✅ 简化的"压入"边界条件（类似凸模下压的近似模拟，但不做真实接触）
- ❌ 不做真实接触 / 摩擦
- ❌ 不做真实的深拉延几何
- ❌ 不做厚度减薄失效分析（这需要 3D 或壳单元）

**这样做的意义**：
- 把问题从"线弹性"升到"弹塑性"，是向冲压成形方向的**关键一步**（塑性变形是冲压的本质）
- 几何+本构+GNN 的整套方法论得到了验证
- 工业界看到这个简化项目会明白："你理解问题，只是没有 LS-DYNA 实操经验——入职 1-2 个月能补上"

**你在面试时要说什么**：

### Day 1 | 弹塑性本构基础

**理论任务**（约 2 小时）：建立概念，不要求推导全部公式。

**最简单的弹塑性模型：完美弹塑性（Perfect Plasticity）**

```
屈服条件：当 von Mises 应力 σ_vm < σ_y（屈服应力），材料是弹性
         当 σ_vm = σ_y，材料开始塑性流动
         σ_vm 不能超过 σ_y（完美弹塑性假设）

应变分解：总应变 = 弹性应变 + 塑性应变
         ε = ε_e + ε_p

应力公式：σ = D · ε_e = D · (ε - ε_p)

塑性流动规则：ε_p 的增量方向沿应力梯度
           dε_p = dλ · ∂f/∂σ   （f 是屈服函数）
```

**稍复杂但更实用：带硬化的弹塑性**

完美弹塑性 + 硬化规律（材料屈服后 σ_y 随塑性应变增大）：
```
σ_y(ε_p_eq) = σ_y_0 + H · ε_p_eq   （线性硬化）
或
σ_y(ε_p_eq) = σ_y_0 + K · (ε_p_eq)^n   （幂指数硬化，冲压用）
```

**写笔记回答**：
- 为什么冲压成形是塑性问题？（回答：板料变形量远超弹性极限）
- 塑性变形后卸载会发生什么？（回答：回弹——弹性应变恢复，塑性应变保留）
- 真实金属的 σ-ε 曲线长什么样？（网上搜"铝合金应力应变曲线"，画一张放笔记里）

### Day 4 | 降级方案：非线性弹性代替塑性

**如果 Day 2-3 的弹塑性求解器做不出来**（这是完全可能的事情），用**非线性弹性**代替：

```python
def sigma_nonlinear_elastic(epsilon, E_0=70e3, alpha=0.5):
    """
    非线性弹性本构（伪塑性）：
      σ = E_0 · ε · (1 - α·|ε|)
    
    不是真实塑性（没有塑性应变累积），但有"应变越大刚度越低"的非线性，
    行为上类似塑性硬化的效果，且数值上是线性问题（不需要 Newton-Raphson）。
    """
    # 应变张量转标量（用等效应变）
    eps_eq = ufl.sqrt(ufl.inner(epsilon, epsilon))
    
    # 变系数的 D 矩阵
    factor = 1 - alpha * eps_eq
    coef = E_0 * factor / (1 - nu**2)
    
    stress = coef * (ufl.tr(epsilon) * ufl.Identity(2) + 
                     2 * mu_adjusted * epsilon)
    return stress
```

**这个降级方案的诚实定位**：
- ✅ 保留了"非线性本构"的数学结构
- ✅ 比线弹性更贴近塑性问题的 AI 挑战（网络需要学非线性映射）
- ❌ **不能说**这是"塑性问题"——这是"非线性弹性"
- ✅ 可以说："我用了非线性弹性本构作为向塑性方向的第一步扩展"

### Day 1 | 设计几何和加载工况

**典型的简化冲压几何**：
- 一块 2D 板料（长 × 宽 = 2.0 × 1.0）
- 中间一个圆孔或方孔（模拟凹模型腔）
- 左右两端固定（模拟压边约束）
- 顶部施加垂直向下的载荷（模拟凸模下压）

**为什么这个几何像冲压**：
- 固定边界 = 压边圈压住板料
- 向下的载荷 = 凸模下压
- 中间的孔 = 凹模开口处

**为什么这个仍然是简化版**：
- 真实冲压凸模是"刚体接触"，不是均匀分布力
- 真实冲压凸模会逐步向下移动，不是瞬间施加
- 真实冲压板料会被"拉"进凹模型腔，产生深拉延

## Week 36: PhyFENet 在冲压数据上的训练（复用 + 改造周）

**本周定位**：把 Part F 的训练体系应用到新的弹塑性数据集上。这**不是**简单的"换数据路径"——有三处必须改造的地方：

1. **归一化**：新数据集的量级和分布和 Part F 不同，`GraphZScoreNormalizer` 必须在新数据上重新 fit
2. **本构 Loss**：塑性场景下 `σ = D·ε` 不再成立，本构 Loss 必须重新设计（见 Day 2-3）
3. **多级网络**：如果 Week 34 选了路径 A（真塑性），解析应力函数失效，必须训练 StressSubNet

**什么是"复用"**：
- ✅ PhyFENet_WithElement_Batched 模型结构
- ✅ batch 训练框架（block-diagonal 聚合矩阵）
- ✅ 训练稳定性措施（梯度裁剪、lr 调度）
- ✅ 评估指标计算逻辑

**什么是"改造"**：
- ⚠️ 本构 Loss 的数学形式
- ⚠️ 多级网络的组装方式（是否需要 StressSubNet）
- ⚠️ 归一化的重新 fit
- ⚠️ 误差分析指标（弹性区 vs 塑性区分开统计）

**心理预期**：本周的工作量介于 Part F Week 30（从零做物理约束实验）和 Part F Week 29（纯工程实现 batch）之间——不是从零，但也不是一键切换。**预估占用 5-6 天，不是 3 天**。

- `GraphZScoreNormalizer` 重新 fit（新数据集，量级可能不同）

### Day 2–3 | 对比实验：纯数据 vs 物理约束

**沿用 Part F Week 30 的实验设计**，在新数据集上重跑 4 组对比：
- A: 纯数据
- B: + 几何一致性 Loss
- C: + 本构 Loss
- D: 全部

**重点观察**：
- **塑性问题的本构更复杂**（不再是 `σ = D·ε` 这种简单线性关系），本构 Loss 可能需要重新设计
- 直接用线弹性的胡克定律作为本构 Loss 参考会是**不严谨的**（但可以作为"弹性阶段的参考"）

**本构 Loss 在塑性场景下的处理**（两种选择）：

**选择 1**：只在弹性阶段应用本构 Loss
```python
# 通过累计塑性应变判断是否处于弹性
is_elastic = (eps_p_eq < threshold)   # threshold 约 1e-5
L_const = mse(
    sigma_pred[is_elastic],
    hookean(strain_pred[is_elastic], E, nu)
)
```

**选择 2**：引入"广义本构 Loss"——用神经网络学塑性本构

这就是 Part F Week 31 的 **StressSubNet** 真正起作用的时候——因为塑性没有解析公式，神经网络是有意义的。这里如果做了进阶线的 StressSubNet，能发挥优势。

## ⚠️ Part G 的 Stop Line（封顶标准）

**Part G 和 Part F 不一样**。Part F 的产出是技术上的——实验结果、评估指标——完成标准可以用数字衡量。Part G 的产出是**就业导向的**——作品集、简历、博客——这种产出**本质上永远可以再打磨**。README 可以再精致、Demo GIF 可以再录一版、博客可以再多写 500 字。

**没有 Stop Line 的后果**：学习者陷入"还不够好"的完美主义陷阱，**拖延进入第三阶段**（面试投递期）。

**Stop Line 定义**：以下 6 条都达到时，**强制进入第三阶段，不允许继续打磨 Part G**。

### Minimum Viable Portfolio (MVP) 标准

- [ ] **简历**：一份 PDF 简历，能让一个陌生技术 HR 在 30 秒内理解你的定位和核心项目
- [ ] **GitHub README**：项目主 README 有：一句话简介 / 技术栈 / 核心结果（至少 1 张图 + 3 个数字）/ 如何运行 / 诚实声明
- [ ] **博客**：至少 1 个平台（知乎/CSDN/其他）发表，字数 1500 字以上，有"遇到的坑"章节
- [ ] **讲解稿**：一份 5 分钟的项目讲解稿，能不看稿自然讲出
- [ ] **核心面试 Q&A**：10 个最高频问题的书面答案
- [ ] **冲压项目**：能跑通、有评估结果、README 能说清楚方法和局限

### 达到 Stop Line 的验证方法

**找一个朋友（或家人）做 5 分钟测试**：
1. 给他看你的 GitHub README（不解释）
2. 给他看你的 PDF 简历（不解释）
3. 让他用自己的话告诉你"你做了什么"

如果他能答出"AI + CAE"、"图神经网络做有限元加速"这两个核心——**Stop Line 达到**。

如果他看完后一脸茫然——**继续打磨直到达到 Stop Line**（但只打磨硬需求，不打磨加分项）。

### 过了 Stop Line 的处理

- **不要继续优化 Part G 的细节**——个人主页、LinkedIn、第 15 分钟讲稿、额外的 Demo GIF、更精致的 Banner——这些**全部可以边投简历边做**
- **立刻进入第三阶段**：开始准备面试题、投递简历
- **面试反馈 > 自己打磨**：一次真实面试能告诉你作品集的 80% 问题，自己打磨 1 周都发现不了这些问题

### 为什么这条很重要

绝大多数新手在就业准备阶段的失败，**不是作品集不够好**，而是**作品集永远在改，永远不投**。
在 11 个月的学习计划尾声，学习者很容易陷入"再花一周就完美了"的错觉——但这一周会变成两周、三周。

**对求职来说，一个 80 分的作品集 + 开始投递 > 一个 95 分的作品集 + 晚 1 个月投递**。
