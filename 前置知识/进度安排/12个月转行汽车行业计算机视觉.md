# 🚗 1 年转岗汽车视觉：可执行版路线图 (实战导向)

> **💡 核心理念：**
> 放弃“学院派”的疯狂刷课，拥抱“工程派”的系统落地。不盲目追求听懂所有前沿数学推导，而是把**模型训练、相机标定、系统接入、端侧部署**彻底打通。一年后，你的状态应该是：“我能拿出 3 个像样的车端项目，并讲清楚数据、误差分析与部署取舍。”

---

## 📅 第 0 月（前置补丁）：跨越代码门槛
*适用人群：Python / NumPy / Git 还不熟练，处于“只能看懂但写不出”的阶段。*

*   **📚 主线任务**：Python 脚本编写、NumPy 数组读写、Matplotlib 基础可视化、Git 基础操作。
*   **🎯 核心目标**：把“能看课”变成“能自己动手在本地/Colab跑代码”。
*   **📦 产出要求**：
    *   1 个 `numpy_basics.ipynb`
    *   1 个 GitHub 仓库并完成首次 Commit。

---

## 🧠 第 1-3 月：深度学习地基与调参直觉
*告别黑盒，建立“训坏了知道怎么修”的诊断能力。*

*   **📚 主线资料**：
    *   **吴恩达**：重点看深度学习基础与改善深层神经网络（调参/正则化）。
    *   **李宏毅**：Lecture 1-3 (重点听反向传播、CNN 基础)。
    *   **d2l (动手学深度学习)**：**不线性硬啃**，当作带代码的数学参考书随查随用。
*   **🎯 核心目标**：吃透损失函数、梯度下降、过拟合/欠拟合原理；掌握 PyTorch 最小训练循环；学会看 Loss 曲线判断问题。
*   **💻 实战项目**：
    *   **最小图像分类**：从零跑通 MNIST/CIFAR-10，画出 Train/Val Loss 曲线。
    *   **规范化实验**：加入数据增强、对比 Adam 与 SGD、做消融实验。
*   **📦 产出要求**：输出 `error_analysis.md`（误差分析报告），记录不同参数对结果的影响。

---

## 📐 第 4-5 月：经典视觉与几何（汽车视觉的灵魂桥梁）
*不懂标定，在车企干不了活。这是区分“网图调包侠”和“汽车视觉工程师”的分水岭。*

*   **📚 主线任务**：OpenCV 核心基础、相机模型（内外参）、相机标定、投影关系（透视变换）。
*   **🎯 核心目标**：理解物理世界是怎么投影到图像上的，为什么会有畸变，标定在车端为什么是基础设施。
*   **💻 实战项目 (二选一)**：
    *   **传统车道线检测**：透视变换 (IPM) -> 边缘检测 -> 曲线拟合。
    *   **相机标定实验**：拍摄棋盘格 -> 求内参矩阵 -> 畸变矫正。
*   **💡 避坑指南**：遇到线性代数/矩阵变换卡壳时，**强烈推荐看《视觉SLAM十四讲》的第 2、3 讲**，只看这两讲作为数学补充即可。

---

## 🚦 第 6-8 月：进入汽车感知与真实数据集
*简历去学生味，开始接触行业真实数据格式。*

*   **📚 主线任务**：
    *   2D 目标检测（YOLO 系列、理解 IoU/NMS/mAP）。
    *   语义分割（可行驶区域/车道线，理解 mIoU）。
    *   **学会使用 Waymo Open Dataset 或 nuScenes 数据集**。
*   **🎯 核心目标**：从普通 CV 项目过渡到真实的“多传感器/多视角”汽车感知项目。
*   **💻 实战项目 (二选一)**：
    *   **交通目标检测**：完成 数据处理 -> 训练 -> 评估 -> 误检分析 的全流程。
    *   **可行驶区域/道路分割**：强调场景理解与失败案例（Corner Case）分析。
*   **💡 避坑指南**：Waymo/nuScenes 数据集极大，**千万不要在本地跑全量数据**。学会切出 100 张图的 Mini-dataset，在本地跑通 Pipeline 后，再去云算力平台跑全量。

---

## ⚙️ 第 9-10 月：系统工程与模型部署
*模型不能只活在 Python 脚本里，要能装进车里。*

*   **📚 主线任务**：
    *   **ROS 2 基础**：Node, Topic, Publisher/Subscriber 概念。
    *   **模型部署**：ONNX 导出、TensorRT 基础、推理测速。
*   **🎯 核心目标**：建立“上下游思维”，知道你的感知结果要怎么发给规控团队，知道模型上车为什么要权衡精度与延迟。
*   **💻 实战项目**：
    *   写一个 ROS 2 小节点：订阅图像 Topic，用前几个月训好的模型做推理，发布检测结果。
    *   PyTorch vs ONNX / TensorRT 推理速度（FPS/Latency）对比实验。
*   **💡 避坑指南**：行业部署强依赖 C++。这一阶段**仍以 Python 编写为主，但要做到“能看懂基本的 C++ 推理代码”**即可，不要为了学 C++ 导致整个主线崩盘。

---

## 🏆 第 11-12 月：作品集整理与面试冲刺
*输出倒逼输入，把一年心血转化为 Offer。*

*   **📚 主线任务**：简历打磨、面试八股文、前沿技术科普。
*   **📦 作品集三件套（必须落实到 GitHub 或博客）**：
    1.  **检测/分割项目**：强调误差分析与业务意义。
    2.  **几何/标定项目**：展示对物理传感器的理解。
    3.  **ROS/部署项目**：展示系统工程能力。
*   **🎯 简历叙事（每个项目灵魂 6 问）**：
    任务是什么？数据是什么？模型是什么？指标是什么？最难的问题在哪？你怎么改进的？
*   **💡 拔高建议 (前沿补充)**：花 1-2 周时间看几篇 **BEV (Bird's Eye View) + Transformer** 的科普文章（如 BEVDet, BEVFormer）。面试时能聊懂 2D 怎么转 3D，是极大的加分项。

# ⏱️ 黄金时间分配法则

在这一年的每一周里，请死死守住以下时间分配（建议每天 1.5-2 小时，周末 4 小时）：

*   📖 **50% 学主线**：看课、看文档。
*   🛠️ **30% 敲代码**：写脚本、训模型、跑 Demo。
*   📝 **20% 做复盘**：写博客、画结构图、整理 GitHub 仓库。**（这 20% 决定了你能不能拿到面试机会）**

# 🚫 绝对要避免的 3 个“天坑”

1.  **把 d2l 当成从头到尾必须线性读完的大书**：它是字典，不是小说。用到什么查什么。
2.  **把 Fast.ai 当作唯一主线**：Fast.ai 是用来“建立工程快感”的提速器，用来快速跑通 baseline，但核心理论必须用吴恩达/李宏毅夯实。
3.  **等“学完全套再做项目”**：完美主义是执行力的天敌。第 1 个月的 MNIST 分类也是项目，哪怕很粗糙，也要先让它跑起来！
