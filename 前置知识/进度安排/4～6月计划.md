## Week 14 · 相机模型——图像是怎么来的

### Day 1 ｜ 小孔成像与坐标系

**理论目标**
- [ ] 🧠 L2 理解 ⭐：小孔成像模型 — 3D 世界中的一个点，通过一个小孔，投影到 2D 像平面上
- [ ] 🧠 L2 理解：四个坐标系的关系（这是整个月最重要的认知框架）：

```
世界坐标系 (Xw,Yw,Zw)
    ↓  外参 [R|t] 变换
相机坐标系 (Xc,Yc,Zc)
    ↓  透视投影（除以 Zc）
归一化图像坐标系 (x,y)
    ↓  内参 K 变换
像素坐标系 (u,v)
```

- [ ] 🧠 L2 理解：每一步变换在做什么
  - 外参：把世界中的点转到"以相机为原点"的坐标系
  - 透视投影：3D → 2D，远处的东西看起来小（除以深度 Z）
  - 内参：从物理单位（毫米）转换到像素单位

### Day 2 ｜ 相机内参矩阵 K

**理论目标**
- [ ] 🧠 L3 推导 ⭐：内参矩阵的结构和每个参数的物理含义

```
K = [ fx  0   cx ]
    [ 0   fy  cy ]
    [ 0   0   1  ]
```

  - fx, fy：焦距（单位是像素），决定"缩放"程度
  - cx, cy：主点（光心在像素坐标系的位置），通常接近图像中心
- [ ] 🧠 L2 理解：fx 和 fy 为什么可能不相等？（像素可能不是正方形）
- [ ] 🧠 L2 理解：内参是相机自身的属性，和相机放在哪里无关

**实践目标**
- [ ] 💻 L2 能照写：用 NumPy 构造一个内参矩阵

```python
K = np.array([[fx, 0, cx],
              [0, fy, cy],
              [0,  0,  1]], dtype=np.float64)
```

- [ ] 💻 L2 能照写：给定一个 3D 点 (X,Y,Z)，手动用矩阵乘法算出它在图像上的像素坐标 (u,v)

```python
point_3d = np.array([X, Y, Z])  # 相机坐标系下的点
point_2d_homo = K @ point_3d    # 齐次坐标
u = point_2d_homo[0] / point_2d_homo[2]
v = point_2d_homo[1] / point_2d_homo[2]
```

**自测**：如果 fx=500, cx=320, fy=500, cy=240，一个 3D 点 (1, 0.5, 5) 投影到像素坐标是多少？（答案：u=320+500×1/5=420, v=240+500×0.5/5=290）

### Day 3 ｜ 相机外参 [R|t]

**理论目标**
- [ ] 🧠 L2 理解：外参描述"相机在世界中的位置和朝向"
- [ ] 🧠 L2 理解：R 是 3×3 旋转矩阵（相机朝哪个方向看），t 是 3×1 平移向量（相机在哪里）
- [ ] 🧠 L2 理解：外参是变化的 — 相机移动了，外参就变了；但内参不变


## Week 15 · 透视变换与 IPM（逆透视映射）

### Day 1 ｜ 齐次坐标与变换矩阵

**理论目标**
- [ ] 🧠 L2 理解：齐次坐标 = 在坐标末尾加一个 1，让平移也能用矩阵乘法表示
  - 普通坐标 (x, y) → 齐次坐标 (x, y, 1)
- [ ] 🧠 L2 理解：2D 变换矩阵的层次

| 变换 | 自由度 | 保持什么 | 矩阵大小 |
|------|--------|---------|---------|
| 平移 | 2 | 形状、大小、方向 | 3×3 |
| 刚体（旋转+平移）| 3 | 形状、大小 | 3×3 |
| 仿射 | 6 | 平行线 | 3×3 |
| 透视（单应性）| 8 | 直线（但平行线不再保持） | 3×3 |

- [ ] 🧠 L2 理解：单应性矩阵 H — 描述两个平面之间的投影关系

**实践目标**

```python
def project_point(K, R, t, P_world):
    """将世界坐标系3D点投影到像素坐标"""
    P_cam = R @ P_world + t          # 世界 → 相机
    P_norm = P_cam / P_cam[2]        # 归一化（除以Z）
    P_pixel = K @ P_norm             # 相机 → 像素
    return P_pixel[0], P_pixel[1]    # 返回 (u, v)
```

- [ ] 💻 L4 能魔改：批量投影多个 3D 点并在图像上可视化
- [ ] 💻 L4 能魔改：反向操作 — 给定像素坐标 + 深度 Z，反算 3D 世界坐标

```python
def backproject_pixel(K, R, t, u, v, Z):
    """像素坐标 + 深度 → 世界坐标"""
    P_norm = np.linalg.inv(K) @ np.array([u, v, 1]) * Z
    P_world = np.linalg.inv(R) @ (P_norm - t)
    return P_world
```

### Day 2 ｜ IoU（交并比）

**理论目标**
- [ ] 🧠 L3 推导 ⭐：IoU = 两个框的交集面积 / 两个框的并集面积

```
IoU = Area(A ∩ B) / Area(A ∪ B)
    = Area(A ∩ B) / (Area(A) + Area(B) - Area(A ∩ B))
```

- [ ] 🧠 L2 理解：IoU 的范围 0~1，IoU > 0.5 通常判定为"检测正确"
- [ ] 🧠 L2 理解：IoU 是目标检测中最基础的评价单位，后续所有指标都建立在它之上

**实践目标**
- [ ] 💻 L3 能默写 ⭐：手写 IoU 计算函数

```python
def compute_iou(box1, box2):
    """box 格式: [x1, y1, x2, y2]"""
    x1 = max(box1[0], box2[0])
    y1 = max(box1[1], box2[1])
    x2 = min(box1[2], box2[2])
    y2 = min(box1[3], box2[3])

    inter = max(0, x2 - x1) * max(0, y2 - y1)
    area1 = (box1[2]-box1[0]) * (box1[3]-box1[1])
    area2 = (box2[2]-box2[0]) * (box2[3]-box2[1])
    union = area1 + area2 - inter

    return inter / union if union > 0 else 0
```

**自测**：两个完全重合的框 IoU=?（1.0）完全不重合的框 IoU=?（0.0）

### Day 3 ｜ NMS（非极大值抑制）

**理论目标**
- [ ] 🧠 L2 理解 ⭐：问题 — 一个物体可能被预测出多个重叠的框
- [ ] 🧠 L2 理解：NMS 的逻辑
  1. 按置信度排序所有框
  2. 取置信度最高的框保留
  3. 删除和它 IoU > 阈值 的所有框（这些框大概率是同一个物体的重复检测）
  4. 对剩下的框重复步骤 2–3
- [ ] 🧠 L2 理解：NMS 阈值通常设为 0.5，越小越激进（删得越多）

**实践目标**
- [ ] 💻 L3 能默写 ⭐：手写 NMS

```python
def nms(boxes, scores, iou_threshold=0.5):
    """boxes: list of [x1,y1,x2,y2], scores: list of float"""
    order = sorted(range(len(scores)), key=lambda i: scores[i], reverse=True)
    keep = []
    while order:
        i = order.pop(0)
        keep.append(i)
        order = [j for j in order if compute_iou(boxes[i], boxes[j]) < iou_threshold]
    return keep
```

- [ ] 💻 L2 能照写：可视化 NMS 前后的检测框数量变化

**自测**：如果不做 NMS，会出现什么问题？（答案：一个物体上堆叠大量重复框，影响后续计数和跟踪）

### Day 4 ｜ Precision / Recall / mAP

**理论目标**
- [ ] 🧠 L2 理解：四个基本概念

| 概念 | 含义 | 直觉 |
|------|------|------|
| TP（True Positive） | 正确检测到的目标 | 检到了，确实有 |
| FP（False Positive） | 误检（检到了但其实没有） | 检到了，但其实没有 |
| FN（False Negative） | 漏检（有但没检到） | 有，但没检到 |

- [ ] 🧠 L3 推导：
  - Precision = TP / (TP + FP) — "检测到的里面有多少是对的"
  - Recall = TP / (TP + FN) — "所有真实目标中有多少被检到了"
- [ ] 🧠 L2 理解：Precision 和 Recall 通常此消彼长 — 想多检出就容易误检，想少误检就可能漏检
- [ ] 🧠 L2 理解：AP = Precision-Recall 曲线下的面积；mAP = 所有类别 AP 的平均值
- [ ] 🧠 L2 理解：mAP@0.5 = IoU 阈值 0.5 时的 mAP（常用）；mAP@0.5:0.95 = 多个阈值的平均（更严格）

### Day 2 ｜ 分割评价指标：mIoU

**理论目标**
- [ ] 🧠 L3 推导 ⭐：像素级 IoU（和检测的 IoU 不同！）

```
类别 c 的 IoU = 预测为 c 且真实为 c 的像素数 / (预测为 c 或真实为 c 的像素数)
mIoU = 所有类别 IoU 的平均值
```

- [ ] 🧠 L2 理解：mIoU 和 Pixel Accuracy 的区别
  - Pixel Accuracy：所有像素中预测对了多少（受大面积类别主导）
  - mIoU：每个类别公平计算后取平均（更公平，是分割任务的标准指标）
- [ ] 🧠 L2 理解：为什么 Pixel Accuracy 不好 — 如果 90% 是背景，模型全预测为背景也能 90%

**实践目标**
- [ ] 💻 L3 能默写：手写单类别 IoU 计算

```python
def pixel_iou(pred, target, class_id):
    pred_mask = (pred == class_id)
    target_mask = (target == class_id)
    intersection = (pred_mask & target_mask).sum()
    union = (pred_mask | target_mask).sum()
    return intersection / union if union > 0 else 0
```

- [ ] 💻 L2 能照写：计算所有类别的 IoU 并求平均（mIoU）

```python
# KITTI 标签文件每行格式：
# type truncated occluded alpha bbox(x1,y1,x2,y2) dimensions(h,w,l) location(x,y,z) rotation_y
def parse_kitti_label(label_path):
    objects = []
    with open(label_path) as f:
        for line in f:
            parts = line.strip().split()
            obj = {
                'type': parts[0],
                'bbox': [float(x) for x in parts[4:8]],
                'dimensions': [float(x) for x in parts[8:11]],
                'location': [float(x) for x in parts[11:14]],
            }
            objects.append(obj)
    return objects
```

```python
def kitti_to_yolo(kitti_label_path, img_width, img_height):
    """将 KITTI 标注转为 YOLO 格式"""
    yolo_lines = []
    for obj in parse_kitti_label(kitti_label_path):
        x1, y1, x2, y2 = obj['bbox']
        x_center = ((x1 + x2) / 2) / img_width
        y_center = ((y1 + y2) / 2) / img_height
        w = (x2 - x1) / img_width
        h = (y2 - y1) / img_height
        class_id = CLASS_MAP[obj['type']]
        yolo_lines.append(f"{class_id} {x_center:.6f} {y_center:.6f} {w:.6f} {h:.6f}")
    return yolo_lines
```

```python
class RoadSegDataset(Dataset):
    def __init__(self, img_dir, mask_dir, transform=None):
        self.img_paths = sorted(glob(img_dir + '/*.jpg'))
        self.mask_paths = sorted(glob(mask_dir + '/*.png'))
        self.transform = transform

    def __getitem__(self, idx):
        img = cv2.imread(self.img_paths[idx])
        mask = cv2.imread(self.mask_paths[idx], cv2.IMREAD_GRAYSCALE)
        if self.transform:
            augmented = self.transform(image=img, mask=mask)
            img, mask = augmented['image'], augmented['mask']
        return img, mask.long()
```
