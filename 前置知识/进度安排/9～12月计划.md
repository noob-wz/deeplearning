```
Q: 为什么选 GraphSAGE 而不是 GCN？
A: FEM 每个样本的网格节点数不同，GCN 需要固定节点数，GraphSAGE 支持归纳学习。

Q: 物理约束具体嵌入了什么方程？
A: 嵌入了弹性力学的本构方程——胡克定律，从 GNN 预测的位移算出应变，
   再用本构方程算应力，和 FEM 的真实应力做 MSE 作为额外的 Loss 项。

Q: 小样本为什么物理约束有用？
A: 物理方程提供了额外的"训练信号"——即使只有 10 个标注样本，
   物理约束仍然在每个节点上施加约束，等于用先验知识弥补数据不足。

Q: 最大误差出现在哪里？为什么？
A: 孔边缘的应力集中区域。因为这里梯度变化剧烈，有限个消息传递层
   的感受野可能不足以精确捕捉局部急变。

Q: 如果要预测弹塑性问题怎么办？
A: 本构方程从线弹性（胡克定律）换成弹塑性模型（如幂指数硬化），
   子网络需要学习非线性的应力-应变关系，训练数据也需要包含塑性变形工况。
```

## Week 41 · 技术八股（上）——深度学习 + GNN

### Day 1–2 ｜ 深度学习基础八股

**理论目标**
- [ ] 🧠 L2 理解 + 🗣️ L2 能讲出：每道题 1 分钟口头回答

**Q1：MSE 和 MAE 的区别？什么时候用哪个？**
```
MSE：对大误差惩罚更重（平方），对异常值敏感
MAE：对所有误差平等对待，更鲁棒
回归任务默认用 MSE（梯度更平滑），有异常值时考虑 MAE 或 Huber Loss
```

**Q2：过拟合怎么判断？怎么解决？**
```
判断：Train Loss 低，Val Loss 高
解决（按优先级）：
1. 加数据 / 数据增强
2. 正则化（Weight Decay / Dropout）
3. Early Stopping
4. 减小模型复杂度
```

**Q3：Adam 优化器的核心思想？**
```
结合 Momentum（一阶矩估计）和 RMSprop（二阶矩估计）
对每个参数自适应调整学习率
梯度大的参数步子自动变小，梯度小的步子自动变大
大多数场景的默认首选
```

**Q4：BatchNorm vs LayerNorm？GNN 中用哪个？**
```
BatchNorm：在 batch 维度归一化（同一个特征跨样本求统计量）
LayerNorm：在 feature 维度归一化（同一个样本跨特征求统计量）
GNN 中：推荐 LayerNorm，因为图的 batch 拼接后节点数不均，BatchNorm 统计量不稳定
```

**Q5：为什么需要数据归一化？Z-Score 的公式？**
```
原因：不同特征量纲不同（坐标 mm vs 力 N vs 应力 MPa），不归一化梯度失衡
Z-Score：X̃ = (X − μ) / σ，映射到均值 0 标准差 1
工程仿真中常用全局归一化（用整个训练集的统计量）
```

---

### Day 3–4 ｜ GNN 八股

**Q6：GCN 的核心公式？一句话解释？**
```
H = σ(D̂^(-½) Â D̂^(-½) X W)
一句话：每个节点把自己和邻居的特征加权平均后过线性变换+激活
```

**Q7：消息传递范式的三步？**
```
1. Message：每个邻居生成消息
2. Aggregate：汇总所有邻居消息（sum/mean/max）
3. Update：用汇总结果更新自身特征
```

**Q8：GraphSAGE 和 GCN 的区别？**
```
GCN：用全部邻居、需固定节点数、转导学习
GraphSAGE：可采样邻居、支持可变节点数、归纳学习
工程仿真选 GraphSAGE：因为每个 FEM 样本的网格可能不同
```

**Q9：GNN 的过平滑是什么？怎么解决？**
```
层数太多 → 所有节点特征趋于相同 → 无法区分
原因：每层和邻居"平均"，多层后信息全局扩散
解决：2–4 层就够 + 残差连接 + LayerNorm
```

**Q10：边特征在 GNN 中怎么处理？**
```
标准 GCN/SAGE 不天然支持边特征
方法：自定义 MessagePassing 层，在 message() 中把邻居节点特征和边特征拼接后过 MLP
边特征在工程中很重要（如节点间距离、相对坐标差）
```

## Week 42 · 技术八股（下）——PINN + FEM + 系统设计

### Day 1–2 ｜ PINN 与物理嵌入八股

**Q11：PINN 的核心思想？**
```
用 MLP 近似 PDE 的解，用 autograd 算方程残差，把残差作为 Loss 的惩罚项
Loss = ω_data·L_data + ω_pde·L_PDE + ω_bc·L_BC
不需要标注数据就能求解方程（纯方程驱动）
```

**Q12：autograd.grad() 和 backward() 的区别？**
```
backward()：算 Loss 对 model.parameters 的梯度（用于训练）
autograd.grad()：算任意输出对任意输入的导数（用于物理约束）
PINN 需要 autograd.grad()——例如算 ∂u/∂x 来检查方程是否满足
```

**Q13：多项损失的权重怎么选？**
```
核心难题：不同 Loss 项的量级可能差几个数量级
手动法：让各项 Loss 量级大致相同
自动法：等比例归一化 / 不确定性加权 / GradNorm
实践中常用 ReduceLROnPlateau + 手动调初始权重
```

**Q14：物理约束在什么时候最有价值？**
```
小样本场景——标注数据少时，物理方程提供额外训练信号
外推场景——预测训练参数范围之外的工况
物理一致性——保证输出满足守恒定律等硬约束
```

---

### Day 3–4 ｜ FEM + 力学八股

**Q15：有限元法的核心思想？**
```
把连续体离散成有限个小单元
在每个单元上用简单函数近似物理场
组装全局方程组 K·u = F 并求解
K=刚度矩阵，u=位移，F=载荷
```

**Q16：弹性力学三大方程？**
```
几何方程：ε = ∂u/∂x（应变 = 位移的空间导数）
物理方程：σ = D·ε（应力 = 弹性矩阵 × 应变）— 本构方程
平衡方程：∇·σ + f = 0（内力 + 体力 = 0）
```

**Q17：什么是本构方程？线弹性和弹塑性有什么区别？**
```
本构方程 = 描述材料应力-应变关系的方程
线弹性：σ = E·ε（线性关系，卸载后完全恢复）
弹塑性：超过屈服点后进入塑性变形，卸载后有永久变形
工程中大部分冲压/碰撞问题涉及弹塑性
```

**Q18：mesh-to-graph 的转换逻辑？**
```
节点坐标 + 材料参数 → 图的节点特征
单元连接关系 → 图的边（单元中每两个节点之间建边）
节点间距离/方向 → 图的边特征
FEM 求解结果（位移/应力） → 标签
```

---

### Day 5–6（周末）｜ 场景题 + 系统设计题

**Q19：给你一个新的 FEM 仿真任务，你的技术方案是什么？**
```
1. 数据：明确输入（网格/材料/边界）和输出（位移/应力/应变）
2. 数据集构建：参数化变量 → 拉丁超立方抽样 → 批量 FEM 求解
3. 数据处理：mesh-to-graph + Z-Score 归一化
4. Baseline：先跑纯数据驱动的 GraphSAGE
5. 物理约束：嵌入相关本构方程
6. 评估：MAE / R² / 推理速度 / 物理一致性检查
7. 迭代：误差分析 → 针对性改进
```

**Q20：如果模型精度不够，你怎么排查？**
```
1. 检查数据：有没有 NaN？归一化正确吗？FEM 求解有没有发散？
2. 检查模型：模型容量够吗？消息传递层数足够吗？
3. 检查训练：Loss 收敛了吗？学习率合适吗？有没有过拟合？
4. 检查物理约束：权重太大/太小？物理方程写对了吗？
5. 检查误差分布：误差集中在哪？是全局差还是局部差？
```

**Q21：如果要从 2D 扩展到 3D 问题，需要改什么？**
```
数据：节点坐标从 (x,y) 变成 (x,y,z)，应变从 3 分量变成 6 分量
网格：单元从三角形/四边形变成四面体/六面体
mesh-to-graph：边提取逻辑不变（从单元连接关系提取）
模型：输入/输出维度改变，其他架构基本不变
物理约束：本构方程换成 3D 形式
计算量：显著增大，可能需要 GPU 优化
```

**Q22：PINN 和你的 GNN+物理约束有什么区别？什么时候用哪个？**
```
PINN：无监督求解 PDE，不需要 FEM 数据，每个新工况要重新训练
GNN+物理约束：有监督预测，需要 FEM 训练数据，训好后毫秒级推理

用 PINN：单个/少量工况的求解
用 GNN：需要大量参数扫描/实时预测的场景（如设计优化循环中）
```
